# 역할
너는 20년차 Unity 인터랙션 콘텐츠 전문 시스템 기획자야.
Windows PC 웹캠과 MediaPipe Hands를 활용한 손 추적 인터랙티브 콘텐츠를 다수 기획했고, 빠른 프로토타입 제작에 필요한 시스템 구조를 작고 검증 가능하게 정리할 수 있어.

# 배경
이 프로젝트는 `Assets/Design/콘텐츠기획3.md`를 기준 콘텐츠 기획서로 삼는다.

콘텐츠의 현재 방향은 다음과 같다.
- 프로젝트명: `Hand Play Room`
- 실행 환경: Windows PC
- 입력 방식: PC 웹캠 + MediaPipe Hands
- Unity 입력 정책: 플레이어 조작은 손 추적 기반, 운영자/디버그 보조는 Unity Old Input 기준
- 공간 구성: 귀엽게 꾸며진 단일 3D 방
- 핵심 조작:
  - 왼손: 원형 이동 영역 안에서 이동/시선 조절, 주먹으로 정지
  - 오른손: 중앙 타겟 조준, 핀치로 오브젝트 집기/놓기, 주먹으로 오브젝트 부수기, 손바닥 유지로 인증
- 핵심 체험: 방 안의 큐브, 퍼즐 상자, 별 램프, 쿠키 블록, 버튼 같은 오브젝트에 가까이 가면 자막 가이드가 뜨고, 손짓으로 상호작용한다.

# 요구사항
`Assets/Design/콘텐츠기획3.md`를 바탕으로 `Assets/Design/기획기본구조.md`를 다시 작성해줘.

작성할 문서는 시스템 기획의 기본 구조이며, 이후 핵심 컨셉, 핵심 메커니즘, 기술 명세, Task 문서로 확장하기 위한 기준 문서가 되어야 한다.

## 작성 범위
`Assets/Design/기획기본구조.md`에는 다음 섹션을 포함해줘.

1. 시스템 기획서
   - 프로젝트명
   - 콘텐츠 개요
   - 프로토타입 목표
   - 기준 콘텐츠 기획서 경로

2. 핵심 컨셉
   - 한 줄 정의
   - 지향 경험
   - 사용자 가치
   - 차별 포인트

3. 핵심 메커니즘
   - 손 인식 시작
   - 왼손 이동/시선 제어
   - 오른손 타겟 조준
   - 핀치 기반 오브젝트 집기/놓기
   - 주먹 기반 오브젝트 파괴/버튼 강타
   - 오브젝트 근접 자막 가이드
   - 별 램프 완료/클리어 흐름
   - 운영자 리셋/디버그 보조

4. 기술 명세
   - 시스템별 항목만 우선 정리한다.
   - 각 항목은 나중에 세부 작성이 가능하도록 명확한 이름과 책임을 가진다.
   - 새로 만드는 public 클래스명은 반드시 `UE_` 접두어를 사용한다.

## 기술 명세 기본 항목
기술 명세에는 최소 다음 시스템 항목을 포함해줘.

1. 경험 흐름 및 씬 제어 시스템
   - 후보 클래스: `UE_HandPlayroomFlowController`

2. 웹캠 입력 및 손 추적 시스템
   - 후보 클래스: `UE_WebcamHandTrackingController`
   - MediaPipe Hands 및 `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection` 활용 전제를 명시한다.

3. 손 제스처 판별 시스템
   - 후보 클래스: `UE_HandGestureController`

4. 플레이어 이동 및 시선 제어 시스템
   - 후보 클래스: `UE_HandPlayerController`

5. 오브젝트 상호작용 시스템
   - 후보 클래스: `UE_HandObjectInteractionController`

6. 근접 가이드 및 HUD 피드백 시스템
   - 후보 클래스: `UE_HandPlayroomFeedbackController`

7. 완료 조건 및 방 상태 시스템
   - 후보 클래스: `UE_HandPlayroomProgressController`

8. 운영자 보조 및 디버그 시스템
   - 후보 클래스: `UE_HandPlayroomOperatorController`

## 작성 원칙
- 프로토타입 버전 제작 범위 안에서 작성한다.
- 단일 방, 제한된 오브젝트 수, 짧은 체험 루프를 기준으로 범위를 작게 유지한다.
- Unity New Input System은 사용하지 않는다.
- 플레이어 조작은 MediaPipe Hands 기반으로 설명하고, 키보드 입력은 운영자 보조/디버그 용도로만 둔다.
- 구현 세부 코드가 아니라 시스템 기획 구조를 작성한다.
- 과도한 확장 기능, 멀티플레이, 네트워크, 저장 시스템은 후순위 또는 제외로 명시한다.
- 이후 Task 분리와 체크리스트 작성이 가능하도록 시스템 책임을 분명하게 쓴다.

# 출력 형식
- Markdown 형식으로 작성한다.
- 파일 경로는 `Assets/Design/기획기본구조.md`로 한다.
- 기존 `기획기본구조.md`의 Air DJ 또는 음악 선택/DJ 관련 내용은 남기지 말고, `Hand Play Room` 기준으로 교체한다.
- 작성 완료 후 `Assets/Checklists/기획문서작업체크리스트.md`에 `기획기본구조 작성요구 재작성 검토` 섹션을 추가하거나 갱신해 구현 결과를 반영한다.
