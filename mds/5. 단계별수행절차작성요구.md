# 요구사항
`Assets/Tasks/` 폴더의 Task 문서를 기준으로, Unity에서 실제 실습 및 단계별 구현을 진행하기 위한 절차를 `Assets/Checklists/단계별수행절차.md` 파일에 작성해줘.

## 작성 목표
- 절차서는 코드 작성 순서가 아니라 Unity 씬에서 바로 보이는 결과를 먼저 만드는 순서로 작성한다.
- 사용자가 `단계 실행`이라고 요청하면 스크립트 생성뿐 아니라 `SampleScene`에 GameObject 배치, 컴포넌트 연결, 기본 Inspector 설정, 씬 저장까지 포함하는 것으로 정의한다.
- 프로토타입 제작 흐름상 UI를 먼저 구성하고, 이후 입력/제스처/운영자 제어를 연결하는 순서로 정리한다.

## 필수 산출물
- `Assets/Checklists/단계별수행절차.md`
- 필요 시 `Assets/Tasks/` 하위 Task 문서도 같은 실행 기준에 맞게 갱신한다.
- 필요 시 `Assets/Checklists/Tasks/` 하위 체크리스트도 실제 검증 항목 기준으로 갱신한다.
- `Assets/Checklists/기획문서작업체크리스트.md`의 단계별 수행 절차 작성 여부를 반영한다.

## 단계 구성 기준
1. `0단계`로 씬과 UI 뼈대를 먼저 만든다.
   - `Assets/Scenes/SampleScene.unity`를 기준 씬으로 사용한다.
   - 씬 루트에 `UE_AppRoot`를 만든다.
   - `UE_MainCanvas`를 만든다.
   - `TitlePanel`, `MusicSelectionPanel`, `PlayHudPanel`을 먼저 배치한다.
   - `EventSystem`이 없으면 생성한다.

2. 이후 단계는 UI 우선 흐름으로 정렬한다.
   - 1단계: `UE_ExperienceFlowController`
   - 2단계: `UE_MusicSelectionController`
   - 3단계: `UE_PlayFeedbackController`
   - 4단계: `UE_WebcamTrackingController`
   - 5단계: `UE_GestureDjController`
   - 6단계: `UE_OperatorControlController`

3. 각 단계에는 반드시 다음 내용을 포함한다.
   - 실행할 Task 문서 경로
   - 서브에이전트 요청 예시
   - Unity 실습 절차
   - `SampleScene`에 만들 GameObject 이름
   - 연결할 컴포넌트와 Inspector 참조
   - 체크리스트 갱신 기준
   - Play Mode를 실행하지 않았을 때 미검증으로 남겨야 할 항목

## Task 문서 동기화 기준
- `Assets/Tasks` 하위 각 Task 문서에는 `UI 우선 실행 순서` 섹션을 추가한다.
- 각 Task의 작업 순서에는 `SampleScene` 배치, `UE_AppRoot` 하위 GameObject 생성, 컴포넌트 연결, Inspector 설정, 씬 저장 절차를 포함한다.
- `UE_PlayFeedbackController`는 UI 우선 흐름상 3단계에서 먼저 실행할 수 있도록 `UE_GestureDjController` 미연결 상태에서도 기본 안내 UI를 표시하게 작성한다.
- `UE_GestureDjController` 단계에서는 이미 배치된 `UE_PlayFeedbackController`에 제스처 컨트롤러 참조를 다시 연결하도록 작성한다.

## 체크리스트 작성 기준
- 실제로 확인한 항목만 `[x]`로 체크한다.
- Play Mode를 실제로 돌리지 않은 항목은 반드시 미검증으로 남긴다.
- 씬 Hierarchy에 GameObject가 있어야 완료되는 항목은 스크립트 파일 생성만으로 체크하지 않는다.

## 예시
1. `0단계`에서 `SampleScene`에 `UE_AppRoot`, `UE_MainCanvas`, `TitlePanel`, `MusicSelectionPanel`, `PlayHudPanel`을 만든다.
2. `01_UE_ExperienceFlowController.md`를 서브에이전트로 실행하고, `SampleScene`의 `UE_AppRoot` 아래에 `UE_ExperienceFlowController` GameObject를 만든 뒤 컴포넌트를 붙인다.
3. 의존성 있는 다음 클래스는 구현하지 말고, 필요한 연결 지점만 남긴다.
4. `Assets/Checklists/Tasks`의 대응 체크리스트에는 실제 확인한 항목만 체크한다.
5. Play Mode를 실행하지 않았다면 `Play Mode 미검증`은 그대로 둔다.
