# 역할 
너는 20년차 유니티 게임 개발자야.

# 배경
`Assets/Design/기획기본구조3.md`에 기획서가 작성돼 있어. 

## 요구사항
- 기획서의 '기술 명세'의 각 클래스들을 task 단위로 분리해서 `Assets/Tasks` 폴더에 MD 파일로 만들어줘. 
- 각 task의 구현을 제대로 검증하기 위해 체크리스트를 만들어서 `Assets/Checklists` 폴더에
 MD 파일로 만들어줘.

### Task 분해 원칙
- 반드시 "1 task = 1 class" 원칙을 유지해줘.
- 클래스 책임은 유지하되, task번호는 기술 명세의 코드 나열 순서가 아니라 실제 Unity 씬에서 오브젝트를 만들고 Inspector로 연결하는 작업 순서 기준으로 정해줘.
- 각 task는 하나씩 구현하고 테스트한 뒤 다음 task로 넘어갈 수 있도록 앞 task의 결과가 다음 task의 구성과 연결로 자연스럽게 이어져야 해. 

### 작성 기준
- 모든 task 파일에는 구현순서 번호를 붙여줘.
- 각 task 문서는 개발자가 바로 구현할 수 있게 작성해줘.
- 각 checklist는 실제 검증 가능한 항목 중심으로 작성해줘.
- Play Mode 를 실제로 돌리지 않은 항목을 반드시 미검증으로 남겨줘.
