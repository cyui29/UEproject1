# 핸드 트래킹 3D 가상공간 탈출 게임 콘텐츠 기획서

## 1. 시나리오 기획

### 1-1. 컨셉 및 기획 의도
- 프로젝트명: `Hand Escape Room`
- 장르: Windows PC 카메라 기반 3D 가상공간 탈출 게임
- 핵심 메시지: 별도 컨트롤러 없이 손의 움직임만으로 공간을 탐색하고, 사물을 조작하며, 단서를 해석해 방을 탈출한다.
- 기획 의도:
  - PC 웹캠과 MediaPipe Hands를 활용해 빠르게 검증 가능한 프로토타입형 핸드 트래킹 게임을 제작한다.
  - 왼손은 이동, 오른손은 인터랙션으로 역할을 분리해 초보자도 조작 규칙을 직관적으로 이해하도록 한다.
  - 손을 뻗고, 잡고, 부수고, 맞추는 신체 감각을 3D 퍼즐 해결 경험과 연결한다.
- 프로토타입 목표:
  - 5~8분 안에 3개 방을 통과하는 짧은 탈출 루프를 완성한다.
  - 손 인식, 이동, 오브젝트 조작, 퍼즐 성공 피드백, 최종 탈출 연출을 우선 구현한다.
  - 복잡한 서사보다 명확한 목표, 즉각적인 피드백, 안정적인 조작감을 우선한다.

### 1-2. 타겟 정의
- 메인 타겟:
  - 10대 후반~30대 체험형 게임 이용자
  - 키보드/마우스 조작보다 제스처 기반 인터랙션에 흥미가 있는 사용자
  - 짧은 시간 안에 몰입감 있는 디지털 방탈출을 경험하고 싶은 사용자
- 서브 타겟:
  - 전시, 체험 부스, 학교 프로젝트, 미디어아트 쇼케이스 방문자
  - Unity와 AI 비전 기술을 활용한 프로토타입 데모 관람자
- 타겟 특성:
  - 조작법을 오래 읽기보다 화면 안내와 즉각 반응으로 학습한다.
  - 실패 시 다시 시도할 수 있는 짧은 퍼즐 구조를 선호한다.
  - 자신의 손동작이 3D 공간 안에서 직접 결과를 만드는 경험에 높은 몰입을 느낀다.

### 1-3. 세계관과 진행 시나리오
- 배경:
  - 사용자는 폐쇄된 실험형 가상공간 `Memory Vault`에 갇힌 테스트 참가자다.
  - 공간은 손의 움직임을 인증 수단으로 사용하며, 왼손과 오른손의 역할을 분리한 보안 퍼즐을 통과해야 출구가 열린다.
- 진행 구조:
  - 튜토리얼 로비: 손 인식, 왼손 이동, 오른손 선택/잡기 학습
  - 1번 방 `Calibration Hall`: 왼손 이동과 정지, 오른손 핀치로 에너지 큐브 배치
  - 2번 방 `Archive Chamber`: 손짓으로 단서 패널 회전, 잠금 문양 정렬
  - 3번 방 `Core Break Room`: 주먹 제스처로 균열 코어 파괴, 최종 문 개방
  - 탈출 연출: 출구 문이 열리고 손바닥 인증으로 게임 클리어
- 승리 조건:
  - 3개 방의 핵심 퍼즐을 순서대로 해결하고 최종 출구 패널에 오른손 손바닥을 2초간 유지한다.
- 실패/재시도 조건:
  - 제한 시간은 프로토타입에서는 선택 사항으로 둔다.
  - 손 인식 실패, 퍼즐 오입력, 오브젝트 낙하 시 즉시 리셋 버튼 또는 자동 재배치로 복구한다.

### 1-4. 사용자 인터랙션
기본 원칙은 `왼손 = 이동/자세`, `오른손 = 선택/조작`, `양손 = 특수 인증 또는 리셋`이다. 입력은 Unity Old Input 보조키를 운영자 디버그용으로만 사용하고, 플레이어 조작은 MediaPipe Hands 기반 손 랜드마크를 기준으로 한다.

| 손 | 손짓/행동 | 인식 기준 | 게임 기능 | 화면 피드백 |
| --- | --- | --- | --- | --- |
| 왼손 | 손바닥 펼치기 | 손바닥 랜드마크가 안정적으로 감지됨 | 이동 모드 활성화 | 왼쪽 HUD에 이동 아이콘 점등 |
| 왼손 | 손바닥을 화면 위쪽으로 이동 | 손 중심점 Y값 상승 | 전진 이동 | 바닥 경로 라인이 앞으로 밝아짐 |
| 왼손 | 손바닥을 화면 아래쪽으로 이동 | 손 중심점 Y값 하강 | 후진 또는 감속 | 경로 라인이 짧아지고 속도 게이지 감소 |
| 왼손 | 손바닥을 좌우로 이동 | 손 중심점 X값 변화 | 좌우 회전 또는 스트레이프 | 화면 중앙 방향 화살표가 좌우로 기울어짐 |
| 왼손 | 주먹 쥐기 | 손가락 끝이 손바닥 쪽으로 접힘 | 이동 정지/고정 | 발밑 고정 링 표시 |
| 왼손 | 손바닥 빠르게 위로 튕기기 | 짧은 시간 내 Y값 급상승 | 짧은 점프 또는 단차 넘기 | 캐릭터 그림자와 점프 라인 표시 |
| 오른손 | 검지로 가리키기 | 검지 끝 방향 벡터 추정 | 포인터 조준 | 조준 레이와 대상 윤곽선 표시 |
| 오른손 | 엄지-검지 핀치 | 두 손가락 끝 거리 임계값 이하 | 오브젝트 잡기/이동 | 잡힌 물체에 시안색 외곽선 표시 |
| 오른손 | 핀치 해제 | 두 손가락 끝 거리 증가 | 오브젝트 놓기/배치 | 배치 가능 위치에 스냅 이펙트 표시 |
| 오른손 | 주먹 쥐기 | 손가락 접힘 상태 유지 | 약한 파괴/버튼 강타 | 균열, 파편, 진동 피드백 |
| 오른손 | 손목 좌우 회전 | 손바닥 방향 벡터 회전 | 다이얼/문양 회전 | 원형 게이지가 회전값에 맞춰 변함 |
| 오른손 | 손바닥 2초 유지 | 특정 패널 앞에서 정지 | 인증/확정 | 인증 원이 차오른 뒤 완료음 재생 |
| 양손 | 양손 펼치고 2초 유지 | 양손 모두 감지, 손바닥 상태 | 퍼즐 리셋 또는 힌트 호출 | 중앙에 힌트 팝업 표시 |

- 조작 보정:
  - 손 중심 좌표는 0.15초 내외의 스무딩을 적용해 떨림을 줄인다.
  - 핀치, 주먹, 손바닥 인증은 최소 유지 시간을 둬 오입력을 줄인다.
  - 손이 화면 밖으로 나가면 이동을 자동 정지하고, 오른손 잡기 상태는 안전 해제한다.

## 2. 콘텐츠 기획

### 2-1. 작품 라인업 및 스펙
| 구분 | 항목 | 프로토타입 스펙 | 메타데이터/비고 |
| --- | --- | --- | --- |
| 엔진 | Unity | Windows PC 빌드, 3D 실시간 렌더링 | 프로젝트 기준 엔진 버전 유지 |
| 입력 | PC 웹캠 | 내장 카메라 또는 USB 웹캠, 720p~1080p, 30fps 이상 | 권장 카메라 위치: 모니터 상단 중앙 |
| 핸드 트래킹 | MediaPipe Hands | 양손 21개 랜드마크, 좌/우 손 구분 | 패키지 import 완료 전제 |
| 입력 방식 | Old Input 보조 | 운영자 리셋, 디버그 토글, 임시 키 입력 | 플레이어 기본 조작은 손 트래킹 |
| 3D 공간 | 탈출 방 3종 | 로비, Calibration Hall, Archive Chamber, Core Break Room | Low-poly 또는 단순 PBR 프롭 권장 |
| UI | UGUI 또는 UI Toolkit | 손 인식 상태, 제스처 가이드, 목표 안내, 힌트 팝업 | 1920x1080 기준 우선 제작 |
| 이미지 | 콘셉트 PNG, UI 아이콘 PNG, 퍼즐 심볼 PNG | `Assets/Concept`, `Assets/Art/UI`, `Assets/Art/Textures` | PNG, 투명 배경 필요 시 RGBA |
| 영상 | 선택 사항 | 인트로/클리어 짧은 MP4 | 프로토타입에서는 Timeline 연출로 대체 가능 |
| 오디오 | 효과음/환경음/BGM | 문 개방, 핀치 선택, 퍼즐 성공, 경고, 코어 파괴 | WAV 또는 OGG, -6dB 피크 기준 |
| 텍스트 | 도슨트/월 텍스트/팝업 | 방별 목표 안내와 힌트 | 한국어 우선, 짧은 문장 |

#### 에셋 메타데이터 기준
- 이미지:
  - 형식: PNG
  - 콘셉트 이미지 권장 해상도: 1920x1080
  - UI 아이콘 권장 해상도: 512x512 또는 1024x1024
  - 파일명 규칙: `concept_`, `ui_`, `symbol_` 접두어 사용
  - 태그: `concept`, `gesture`, `room`, `ui`, `puzzle`
- 오디오:
  - 형식: WAV 또는 OGG
  - 샘플레이트: 44.1kHz 이상
  - 루프 BGM 길이: 60~90초
  - 효과음 길이: 0.2~3초
  - 태그: `sfx_interaction`, `sfx_success`, `sfx_error`, `ambience`, `bgm`
- 영상:
  - 형식: MP4(H.264)
  - 해상도: 1920x1080
  - 용도: 인트로, 클리어, 대기 화면
  - 프로토타입 우선순위: 낮음

### 2-2. 공간 테마 및 방 구조
- 전체 공간 콘셉트: 손동작 보안 시스템이 설치된 미래형 기억 보관소
- 컬러 팔레트:
  - 기본: `#101820` 딥 블랙, `#263238` 그래파이트
  - 인터랙션: `#18D6C3` 시안 그린, `#F6C85F` 앰버
  - 위험/오류: `#E94F37` 레드 오렌지
  - 클리어: `#F5F7FA` 화이트 라이트
- 조명 방향:
  - 이동 가능 경로는 낮은 바닥 라이트로 안내한다.
  - 조작 가능한 오브젝트는 오른손 포인터가 닿을 때 외곽선과 소프트 글로우를 표시한다.
  - 방 클리어 시 색온도가 차갑고 어두운 톤에서 밝은 화이트 톤으로 전환된다.

| 구역 | 목적 | 방 구조 | 핵심 퍼즐 | 배치 요소 |
| --- | --- | --- | --- | --- |
| 튜토리얼 로비 | 손 인식과 기본 조작 학습 | 직선 복도와 넓은 시작 발판 | 왼손 이동, 오른손 핀치 연습 | 카메라 안내 패널, 손 실루엣 HUD, 연습용 큐브 |
| 1번 방 Calibration Hall | 이동과 잡기 검증 | 중앙 기둥 1개, 좌우 받침대 2개 | 에너지 큐브를 올바른 받침대에 배치 | 큐브, 받침대, 잠긴 문, 바닥 경로 |
| 2번 방 Archive Chamber | 회전 조작과 단서 해석 | 원형 방, 벽면 기록 패널 4개 | 문양 다이얼을 월 텍스트 힌트와 맞춰 정렬 | 회전 다이얼, 기록 패널, 힌트 조명 |
| 3번 방 Core Break Room | 파괴와 인증 | 좁은 진입로와 중앙 코어 | 오른손 주먹으로 균열 코어를 3회 파괴 후 손바닥 인증 | 코어, 방어막, 최종 출구, 클리어 라이트 |

#### 방별 진행 흐름
- 튜토리얼 로비:
  - 화면 중앙에 서서 양손을 보여준다.
  - 왼손 손바닥을 위로 올려 전진하고, 주먹으로 멈춘다.
  - 오른손 핀치로 연습 큐브를 집어 지정 위치에 놓는다.
- 1번 방:
  - 왼손으로 이동해 큐브 근처로 접근한다.
  - 오른손 핀치로 큐브를 들어 색이 맞는 받침대에 배치한다.
  - 두 큐브가 모두 맞으면 문이 열린다.
- 2번 방:
  - 벽의 문장과 심볼을 읽고 올바른 순서를 추론한다.
  - 오른손 손목 회전으로 다이얼을 돌린다.
  - 정렬 성공 시 아카이브 문이 열린다.
- 3번 방:
  - 중앙 코어의 붉은 균열이 열리는 타이밍에 오른손 주먹을 쥔다.
  - 3회 성공하면 방어막이 해제된다.
  - 최종 패널에 오른손 손바닥을 유지해 탈출한다.

### 2-3. 스토리텔링 및 텍스트

#### 도슨트 스크립트
- 로비:
  - `Memory Vault에 오신 것을 환영합니다. 이 공간은 사용자의 손 움직임을 열쇠로 인식합니다. 왼손으로 이동하고 오른손으로 사물을 조작해 출구를 찾아주세요.`
- 1번 방:
  - `이 방은 움직임과 선택을 검증합니다. 왼손으로 받침대까지 이동하고, 오른손으로 에너지 큐브를 들어 올바른 위치에 놓으세요.`
- 2번 방:
  - `기록은 벽에 남아 있고, 해답은 회전 속에 숨어 있습니다. 오른손을 돌려 문양을 맞추면 다음 방으로 향하는 잠금이 풀립니다.`
- 3번 방:
  - `중앙 코어가 가상공간을 붙잡고 있습니다. 균열이 드러나는 순간 손을 쥐어 코어를 깨고, 마지막으로 손바닥을 인증 패널에 올려 탈출하세요.`

#### 월 텍스트
- 로비 월:
  - `LEFT HAND: MOVE`
  - `RIGHT HAND: INTERACT`
  - `SHOW BOTH HANDS TO START`
- 1번 방 월:
  - `빛은 같은 색의 자리에서 안정된다.`
  - `움직임은 왼손, 선택은 오른손.`
- 2번 방 월:
  - `첫 번째 기록은 문을 보고, 두 번째 기록은 빛을 따른다.`
  - `회전하는 손끝이 잠긴 기억을 맞춘다.`
- 3번 방 월:
  - `균열이 열릴 때만 코어는 약해진다.`
  - `마지막 문은 열린 손을 기억한다.`

#### 팝업 안내문
- 시작:
  - `카메라가 양손을 볼 수 있도록 화면 중앙에 서주세요.`
  - `왼손은 이동, 오른손은 상호작용입니다.`
- 손 인식:
  - `왼손 인식 완료`
  - `오른손 인식 완료`
  - `손이 화면 밖으로 벗어났습니다. 카메라 중앙으로 돌아와 주세요.`
- 이동:
  - `왼손 손바닥을 위로 올리면 앞으로 이동합니다.`
  - `왼손 주먹을 쥐면 멈춥니다.`
- 인터랙션:
  - `오른손 검지로 대상을 가리키세요.`
  - `엄지와 검지를 모아 물체를 잡으세요.`
  - `손을 펴면 물체를 놓습니다.`
  - `주먹을 쥐어 균열 코어를 타격하세요.`
- 성공/실패:
  - `퍼즐 동기화 완료. 문이 열립니다.`
  - `정렬이 맞지 않습니다. 벽의 기록을 다시 확인하세요.`
  - `탈출 인증 완료. Memory Vault가 해제됩니다.`

### 2-4. 기술 사양
| 항목 | 내용 |
| --- | --- |
| 메인 지원 플랫폼 | Windows PC |
| 실행 방식 | Unity Standalone 빌드 |
| 카메라 | Windows 호환 내장 카메라 또는 USB 웹캠 |
| 핸드 트래킹 | MediaPipe Hands, 양손 추적, 21개 랜드마크 |
| 입력 정책 | 플레이어 조작은 핸드 트래킹, 운영자/디버그는 Unity Old Input |
| 권장 해상도 | 1920x1080 |
| 권장 프레임 | 60fps 게임 렌더링, 카메라 입력 30fps 이상 |
| 렌더링 | 3D 실시간 공간, 단순 포스트프로세싱, 외곽선 피드백 |
| UI | 손 인식 상태, 제스처 아이콘, 목표 텍스트, 힌트 팝업 |
| 네트워크 | 필수 아님 |
| 저장 | 프로토타입에서는 로컬 설정값만 선택 저장 |
| 접근성 | 손 인식 실패 시 키보드 운영자 리셋, 힌트 팝업, 입력 유지 시간 완화 옵션 |

#### 구현 우선순위
- 1순위:
  - MediaPipe 양손 인식 연동
  - 왼손 이동, 오른손 포인터/핀치/주먹 판정
  - 1개 방 퍼즐 완성
- 2순위:
  - 3개 방 진행 루프
  - 도슨트/팝업 텍스트와 성공 피드백
  - 오디오 효과와 조명 전환
- 3순위:
  - 콘셉트 이미지 기반 UI/공간 아트 패스
  - 인트로/클리어 영상 연출
  - 난이도 옵션과 힌트 시스템

### 2-5. 콘셉트 이미지 / 무드보드
실제 콘셉트 이미지는 imagegen으로 생성해 `Assets/Concept`에 PNG 파일로 저장한다. 프로토타입 단계에서는 전체 공간 무드와 UI 조작 무드를 먼저 기준 이미지로 사용한다.

| 파일명 | 용도 | 포함 요소 | 활용 가이드 |
| --- | --- | --- | --- |
| `Assets/Concept/hand_escape_room_stage_moodboard.png` | 전체 공간 무드보드 | 3D 탈출 공간, 잠긴 문, 퍼즐 오브젝트, 손 추적 기반 상호작용 분위기 | 기획 공유용 대표 이미지, 첫 화면 분위기와 발표 자료 커버로 사용 |
| `Assets/Concept/hand_escape_room_ui_moodboard.png` | UI/인터랙션 무드보드 | 왼손 이동, 오른손 인터랙션, 손 랜드마크 상태, 목표 HUD, 퍼즐 진행 상태 | 튜토리얼 HUD와 제스처 안내 UI 제작 기준 |

| `Assets/Concept/hand_escape_room_2d_floorplan.png` | 전체 2D 평면도 | 시작 지점부터 최종 출구까지의 방 연결, 진행 동선, 잠금 문, 퍼즐 배치 | Unity 씬 블록아웃과 플레이 동선 검토 기준 |

#### 무드보드 방향
- 시각 분위기:
  - 어두운 실험실 배경에 시안 그린 인터랙션 라이트와 앰버 퍼즐 포인트를 사용한다.
  - 공간은 복잡한 사실 묘사보다 명확한 길, 문, 퍼즐 오브젝트가 잘 보이는 프로토타입형 구성을 우선한다.
- UI 스타일:
  - 손 인식 상태는 좌하단/우하단에 각각 배치해 왼손과 오른손 역할을 분리한다.
  - 제스처 아이콘은 한눈에 읽히는 단색 PNG 아이콘으로 제작한다.
  - 팝업 안내문은 1~2문장 이내로 짧게 표시한다.
- 폰트 스타일:
  - 한글 가독성이 좋은 산세리프체를 사용한다.
  - 제목은 굵게, 안내문은 중간 굵기로 구성한다.
- 조명 효과:
  - 상호작용 가능 대상은 외곽선, 바닥 라이트, 짧은 파티클로 강조한다.
  - 실패 피드백은 붉은 점멸을 짧게 사용하고, 장시간 깜빡임은 피한다.

## 3. 프로토타입 기대 효과
- Windows PC와 웹캠만으로 핸드 트래킹 탈출 게임의 핵심 재미를 빠르게 검증할 수 있다.
- 왼손/오른손 역할 분리가 명확해 손동작 인터랙션 학습 비용을 줄인다.
- 3개 방 구조로 기획, 구현, 테스트 범위를 작게 유지하면서도 완결된 플레이 흐름을 제공한다.
- 향후 방 추가, 퍼즐 추가, 전시형 키오스크 운영, 난이도 조절로 확장하기 쉽다.
