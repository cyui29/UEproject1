# 핸드 트래킹 DJ 인터랙션 콘텐츠 기획서

## 1. 시나리오 기획

### 1-1. 컨셉 및 기획 의도
- 프로젝트명: `Air DJ: 손짓으로 믹스하는 나만의 무대`
- 핵심 메시지: 음악을 잘 몰라도 손짓만으로 리듬과 분위기를 바꾸며, 누구나 즉시 DJ가 되는 즐거움을 경험한다.
- 기획 의도:
  - Windows PC 카메라 기반 핸드 트래킹으로 별도 장비 없이 접근 가능한 인터랙션 콘텐츠를 구현한다.
  - 복잡한 음악 제작 도구를 단순한 손동작으로 치환해 비전공자도 부담 없이 참여하도록 설계한다.
  
- 목적:
  - 짧은 체험 시간 안에 성취감과 재미를 전달한다.
  - 다양한 실내 체험 환경과 이벤트 현장에서 활용 가능한 범용형 음악 인터랙션 포맷을 확보한다.

### 1-2. 타겟 정의
- 메인 타겟:
  - 음악 비전공자
  - 체험형 디지털 콘텐츠 이용자
  - 10대 후반~30대 일반 사용자
  - SNS 촬영과 공유를 즐기는 방문자
- 서브 타겟:
  - 가족 단위 체험객
  - 기업 행사 및 브랜드 체험 공간 방문자
  - 인터랙티브 콘텐츠에 익숙하지 않은 초심자
- 타겟 특성 분석:
  - 조작법을 길게 학습하지 않고 즉시 반응을 기대한다.
  - 실패 부담이 적고, 어떤 동작을 해도 결과가 그럴듯하게 나와야 만족도가 높다.
  - 시각적으로 화려하고, 본인의 제스처가 화면과 음악에 직접 반영되는 피드백을 선호한다.

### 1-3. 사용자 인터랙션
기본 원칙은 `한 손 = 개별 파라미터 제어`, `양손 = 장면 전환급 변화`, `정지 = 유지`, `큰 동작 = 극적인 효과`이다.

| 손짓/행동 | 인식 방식 | 음악 변화 | 화면 반응 |
| --- | --- | --- | --- |
| 손바닥 펼치기 | 손 감지 시작, 트래킹 활성화 | 기본 비트와 배경 패드 재생 시작 | 손 주변 링 이펙트와 웨이브 라인 점등 |
| 오른손 위로 올리기 | Y축 상승 | 필터 오픈, 고역 강조, 에너지 상승 | 우측 파형 밝기 증가, 파티클 상승 |
| 오른손 아래로 내리기 | Y축 하강 | 필터 클로즈, 저역 중심, 차분한 무드 | 우측 파형 밀도 감소, 조명 다운 |
| 왼손 위로 올리기 | Y축 상승 | 볼륨 또는 악기 레이어 추가 | 좌측 패널 점등, 사운드 레이어 카드 활성화 |
| 왼손 아래로 내리기 | Y축 하강 | 볼륨 감소 또는 레이어 뮤트 | 좌측 패널 어두워짐, 레이어 비활성화 |
| 손을 좌우로 이동 | X축 이동 | 크로스페이더 이동, 두 트랙 간 비율 변화 | 중앙 아치형 미터가 좌우로 이동 |
| 양손 벌리기 | 두 손 간 거리 증가 | 리버브/공간감 확대, 드롭 직전 빌드업 | 화면 전체 글로우 확대, 배경 웨이브 확장 |
| 양손 모으기 | 두 손 간 거리 감소 | 리버브 축소, 사운드 집중 | UI가 중앙으로 응집, 입자 수렴 |
| 손목 회전 | 회전 각도 변화 | 스크래치, 필터 스윕, FX 모듈레이션 | 원형 다이얼 회전, 잔상 궤적 표시 |
| 주먹 쥐기 후 펴기 | 상태 전환 | 킥, 스네어, 샘플 원샷 트리거 | 순간 플래시와 비트 펄스 발생 |
| 두 손 1초 정지 | 포즈 유지 | 현재 상태 홀드, 루프 잠금 | 손 외곽선 고정, HOLD 연출 표시 |
| 양손 크게 회전 | 제스처 이벤트 | 곡 섹션 전환 또는 하이라이트 연출 | 3D 오브젝트와 배경 파동이 크게 회전 |

- 난이도 설계:
  - 초급 모드: 볼륨, 필터, 크로스페이더, 효과 1종만 활성화
  - 일반 모드: 레이어 추가, 루프, 빌드업, 드롭 전환 활성화
- UX 설계 포인트:
  - 잘못된 입력도 불쾌한 소리가 나지 않도록, 사전 정의된 스케일과 템포 범위 안에서만 변형한다.
  - 손 인식 시 화면 하단에 직관적인 제스처 아이콘을 항상 노출해 학습 부담을 줄인다.
  - 음악 반응은 0.1~0.2초 이내 피드백을 목표로 해 몰입감을 유지한다.

## 2. 콘텐츠 기획

### 2-1. 작품 라인업(스펙)
| 구분 | 항목 | 내용 |
| --- | --- | --- |
| 엔진 | Unity | Windows PC 실행용 실시간 인터랙션 구현 |
| 카메라 입력 | USB 웹캠 또는 내장 카메라 | 1080p 권장, 30fps 이상 |
| 손 추적 SDK | MediaPipe Hands 또는 유사 손 관절 추적 SDK | 21개 랜드마크 기반 양손 인식 |
| 입력 보정 | OpenCV 또는 Unity 내 후처리 | 노이즈 제거, 스무딩, 손 좌표 정규화 |
| 오디오 엔진 | Unity AudioMixer, FMOD, Wwise 중 택1 | 실시간 믹싱, 필터, 루프, 원샷 트리거 |
| 비주얼 시스템 | Shader Graph, VFX Graph, Timeline | 파형, 입자, 반응형 조명 연출 |
| UI | Unity UI Toolkit 또는 UGUI | 제스처 가이드, 상태 표시, 피드백 HUD |
| 3D/그래픽 | 배경 공간, 반응형 오브젝트, 이펙트 텍스처 | 공간 몰입감과 음악 반응 표현 |
| 오디오 소스 | 장르별 루프팩, 드럼, 베이스, 패드, FX | BPM 통일, 키 충돌 없는 샘플 세트 구성 |
| 영상/이미지 | 인트로, 대기화면, 도움말 패널, 콘셉트 이미지 | 운영 및 브랜딩용 시각 자료 |

#### 권장 메타데이터
- 이미지:
  - 파일 형식: PNG
  - 권장 해상도: 1920x1080 이상
  - 용도 태그: `space`, `ui`, `guide`, `promo`
- 오디오:
  - 파일 형식: WAV 또는 고품질 OGG
  - BPM: 100 / 110 / 120 중 통일
  - 키: C minor 또는 A minor 중심의 충돌 적은 구성
  - 길이: 4, 8, 16마디 루프 단위
- 영상:
  - 파일 형식: MP4(H.264)
  - 용도: 인트로, 튜토리얼, 대기 연출

### 2-2. 공간 테마 및 배치
- 공간 콘셉트: `미래형 사운드 랩 + 인터랙티브 퍼포먼스 스테이지`
- 공간 무드:
  - 메인 컬러: 딥 차콜, 틸, 시안
  - 포인트 컬러: 웜 오렌지, 코랄
  - 조명: 사용자의 손 위치와 음악 강약에 반응하는 간접 조명
- 권장 섹션 구성:

| 섹션 | 역할 | 배치 요소 | 색상/조명 모드 |
| --- | --- | --- | --- |
| Entrance | 대기 및 유입 | 타이틀 월, 체험 방법 요약, 대기선 | 어두운 배경 + 은은한 틸 라이트 |
| Tutorial Zone | 제스처 학습 | 손 아이콘, 모션 예시, 카메라 위치 안내 | 밝은 시안 계열, 명확한 정보 조명 |
| Play Zone | 메인 체험 | 대형 디스플레이, 카메라, 스탠딩 포지션, 메인 오디오 | 틸/오렌지 대비, 반응형 조명 강화 |
| Highlight Zone | 결과 감상 | 하이라이트 리플레이, 캡처 화면, 베스트 모먼트 | 코랄 포인트, 여운 있는 조명 |

- 배치 원칙:
  - 카메라는 디스플레이 상단 중앙에 위치해 사용자의 양손이 모두 안정적으로 잡히게 한다.
  - 사용자의 스탠딩 위치는 화면과 1.8m~2.5m 거리로 표시한다.
  - 좌우 측면 구조물은 빛 반사와 프레이밍 효과를 주되, 카메라 시야를 방해하지 않게 구성한다.

### 2-3. 스토리텔링 및 텍스트
#### 작품 설명
`Air DJ`는 손짓만으로 음악의 분위기를 바꾸는 인터랙티브 사운드 퍼포먼스입니다. 사용자는 복잡한 장비나 전문 지식 없이도 손을 올리고, 벌리고, 회전하는 간단한 동작만으로 리듬과 효과를 조절할 수 있습니다. 화면 속 파형과 빛은 사용자의 움직임에 실시간으로 반응하며, 사용자는 즉석에서 자신만의 퍼포먼스를 완성하게 됩니다.

#### 메인 텍스트
- 메인 카피: `손끝으로 리듬을 움직이다`
- 서브 카피: `카메라 앞에 서는 순간, 당신의 제스처가 음악이 됩니다.`

#### 팝업 안내문
- 대기 화면:
  - `화면 중앙에 서서 두 손을 들어주세요.`
  - `손이 인식되면 사운드가 시작됩니다.`
- 플레이 중:
  - `손을 위로 올리면 사운드가 더 밝아집니다.`
  - `양손을 벌리면 공간감이 확장됩니다.`
  - `손을 좌우로 움직여 두 사운드를 섞어보세요.`
- 오류/예외:
  - `손이 화면에서 벗어났습니다. 중앙으로 다시 이동해주세요.`
  - `주변 조명이 너무 어두우면 인식률이 낮아질 수 있습니다.`

### 2-5. 콘셉트 이미지 / 무드보드
- 역할:
  - 인터랙션하는 사용자 모습, 화면 UI 분위기, 색감, 조명 연출 방향을 한눈에 공유하는 비주얼 레퍼런스
- 포함 요소:
  - 체험 공간 레퍼런스
  - 화면 UI 및 제스처 피드백 연출
  - 권장 색상 팔레트: `#0F1720`, `#12D7E5`, `#59F0FF`, `#FF9A4D`, `#FF6B57`
  - 폰트 스타일 방향: 굵고 가독성 높은 산세리프, 미래적이되 지나치게 기계적이지 않은 인상
  - 조명 효과: 소프트 글로우, 파형 기반 라인 라이트, 음악 반응형 입자 연출
- 생성 파일:
  - `Assets/Concept/hand_dj_space_moodboard.png`
  - `Assets/Concept/hand_dj_ui_moodboard.png`
- 활용 가이드:
  - `hand_dj_space_moodboard.png`: 체험 공간의 사용 장면, 사용자 포즈, 조명 무드, 디스플레이 스케일 참고
  - `hand_dj_ui_moodboard.png`: 손 제스처 가이드 UI, 믹싱 HUD, 색상 밸런스, 시각 피드백 구조 참고

## 기대 효과
- 초보자도 즉시 성과를 느낄 수 있는 참여형 음악 콘텐츠로 체류 시간을 늘린다.
- 손짓과 음악, 빛이 결합된 퍼포먼스형 경험으로 사용 장면 자체가 강한 홍보 포인트가 된다.
- 향후 장르별 테마 스킨, 교육형 모드, 멀티 유저 모드 등으로 확장 가능한 베이스 포맷을 확보한다.
