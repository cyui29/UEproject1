# 귀여운 단일 방 핸드 트래킹 3D 인터랙션 콘텐츠 기획서

## 1. 시나리오 기획

### 1-1. 컨셉 및 기획 의도
- 프로젝트명: `Hand Play Room`
- 장르: Windows PC 카메라 기반 3D 손동작 인터랙션 콘텐츠
- 핵심 메시지: 컨트롤러 없이 왼손으로 귀여운 방 안을 이동하고, 오른손으로 오브젝트를 집고 부수며 반응을 발견한다.
- 기획 의도:
  - PC 웹캠과 MediaPipe Hands를 활용해 손 추적 기반 이동, 조준, 잡기, 파괴를 빠르게 검증한다.
  - 복잡한 방탈출 구조보다 한 개의 방에서 조작 학습과 인터랙션 재미를 명확하게 보여준다.
  - 귀여운 장난감 방 분위기와 즉각적인 자막/이펙트 피드백으로 초보 사용자도 부담 없이 체험하게 한다.
- 프로토타입 목표:
  - 3~5분 안에 손 인식, 이동, 오브젝트 근접 안내, 핀치 잡기, 주먹 파괴를 모두 경험한다.
  - `왼손 = 이동/시선`, `오른손 = 조준/상호작용` 규칙을 화면 UI와 공간 반응으로 학습시킨다.
  - 구현 범위는 단일 씬, 단일 방, 상호작용 오브젝트 5~7개로 제한한다.

### 1-2. 타겟 정의
- 메인 타겟:
  - 10대 후반~30대 체험형 게임 및 인터랙티브 콘텐츠 이용자
  - 키보드/마우스보다 몸짓 기반 조작에 호기심이 있는 사용자
  - 전시 부스나 수업 프로젝트에서 짧게 체험할 수 있는 콘텐츠가 필요한 사용자
- 서브 타겟:
  - Unity, MediaPipe, 컴퓨터 비전 기반 프로토타입을 검토하는 개발자와 기획자
  - 어린 사용자도 이해할 수 있는 직관적인 제스처 콘텐츠를 원하는 관람자
- 타겟 특성:
  - 긴 설명보다 화면 안의 아이콘, 자막, 오브젝트 반응으로 조작을 익힌다.
  - 실패해도 바로 다시 시도할 수 있는 짧은 액션을 선호한다.
  - 자신의 손이 방 안 물건을 직접 움직인다는 느낌에 흥미를 느낀다.

### 1-3. 세계관과 진행 시나리오
- 배경:
  - 사용자는 작은 장난감 공방 `손짓 놀이방`에 들어온 체험자다.
  - 방 안의 장난감과 소품은 손동작을 알아보고, 가까이 다가가면 말을 걸듯 짧은 가이드를 띄운다.
  - 목표는 방 안의 주요 오브젝트를 모두 한 번씩 상호작용해 `반짝 별 램프`를 켜는 것이다.
- 진행 구조:
  - 시작: 카메라 앞에서 양손을 보여주면 손 인식 UI가 켜지고 방 중앙에 플레이어가 배치된다.
  - 이동 학습: 왼손을 원형 이동 영역 안에서 움직여 방 안의 장난감 테이블 근처로 이동한다.
  - 잡기 학습: 오른손 포인터로 큐브를 조준하고 핀치로 집어 1m 앞 프리뷰 위치로 가져온다.
  - 배치 학습: 핀치를 풀어 색이 맞는 매트 위에 큐브를 놓는다.
  - 파괴 학습: 깨지는 쿠키 블록 앞에서 오른손 주먹을 쥐어 블록을 부순다.
  - 완료: 주요 상호작용 3개 이상을 성공하면 별 램프가 켜지고 짧은 클리어 연출이 나온다.
- 승리 조건:
  - `큐브 집기`, `큐브 배치`, `쿠키 블록 부수기`, `별 램프 인증` 중 3개 이상을 완료한다.
- 재시도 조건:
  - 손이 화면 밖으로 나가면 이동과 잡기 상태를 안전 해제한다.
  - 오브젝트가 바닥 밖으로 떨어지면 시작 위치로 자동 복귀한다.
  - Old Input 기반 디버그 키로 운영자가 리셋할 수 있다.

### 1-4. 사용자 인터랙션
기본 원칙은 `왼손 = 이동과 시선`, `오른손 = 오브젝트 조준과 상호작용`이다. 플레이어 조작은 MediaPipe Hands 랜드마크를 기준으로 판정하고, Unity Old Input은 운영자 디버그와 리셋 보조에만 사용한다.

| 손 | 손짓/행동 | 인식 기준 | 게임 기능 | 화면 피드백 |
| --- | --- | --- | --- | --- |
| 왼손 | 손바닥 펼치기 | 왼손 21개 랜드마크 안정 감지 | 이동 모드 활성화 | 왼쪽 하단 이동 원 UI 점등 |
| 왼손 | 원 안에서 왼쪽 이동 | 손 중심점 X값이 기준선보다 왼쪽 | 좌측 이동 또는 좌회전 | 이동 원의 왼쪽 영역이 민트색으로 밝아짐 |
| 왼손 | 원 안에서 오른쪽 이동 | 손 중심점 X값이 기준선보다 오른쪽 | 우측 이동 또는 우회전 | 이동 원의 오른쪽 영역이 민트색으로 밝아짐 |
| 왼손 | 원 안에서 위로 이동 | 손 중심점 Y값 상승 | 카메라가 위쪽을 응시하거나 전진 | 화면 중앙 시선 화살표가 위로 기울어짐 |
| 왼손 | 원 안에서 아래로 이동 | 손 중심점 Y값 하강 | 카메라가 아래쪽을 응시하거나 후진 | 화면 중앙 시선 화살표가 아래로 기울어짐 |
| 왼손 | 주먹 쥐기 | 손가락 끝이 손바닥 쪽으로 접힘 | 이동 정지 | 플레이어 발밑 정지 링 표시 |
| 오른손 | 검지로 가리키기 | 검지 끝 방향 벡터 추정 | 중앙 타겟 십자와 조준 레이 활성화 | 오브젝트 외곽선 표시 |
| 오른손 | 엄지-검지 핀치 | 두 손가락 끝 거리 임계값 이하 | 오브젝트 집기 | 오브젝트가 플레이어 앞 약 1m 위치로 이동 |
| 오른손 | 핀치 유지 | 핀치 상태 지속 | 잡은 오브젝트를 조준점 앞에 유지 | 잡기 상태 아이콘과 시안색 연결선 표시 |
| 오른손 | 핀치 해제 | 두 손가락 끝 거리 증가 | 오브젝트 놓기/배치 | 배치 가능 지점에서 별 파티클 표시 |
| 오른손 | 주먹 쥐기 | 손가락 접힘 상태 유지 | 약한 파괴 또는 버튼 강타 | 깨짐 이펙트, 짧은 화면 진동 |
| 오른손 | 손바닥 2초 유지 | 특정 패널 앞에서 손바닥 정지 | 확인/인증 | 원형 게이지가 차오른 뒤 완료음 재생 |
| 양손 | 양손 펼치고 2초 유지 | 양손 모두 감지 | 힌트 호출 또는 방 리셋 | 중앙 힌트 팝업 표시 |

- 조작 보정:
  - 손 중심 좌표에는 약 0.15초 스무딩을 적용해 떨림을 줄인다.
  - 핀치와 주먹은 최소 0.2초 유지 후 입력으로 인정한다.
  - 화면 좌우를 반으로 나눠 왼손 이동 영역과 오른손 조준 영역을 시각적으로 분리한다.
  - 오른손 잡기 중 손이 사라지면 오브젝트를 현재 위치에 놓고, 1초 뒤 안전 위치로 스냅한다.

## 2. 콘텐츠 기획

### 2-1. 작품 라인업 및 스펙
| 구분 | 항목 | 프로토타입 스펙 | 메타데이터/비고 |
| --- | --- | --- | --- |
| 엔진 | Unity | Windows PC Standalone, 3D 실시간 렌더링 | 기존 프로젝트 설정 유지 |
| 입력 | PC 웹캠 | 내장 카메라 또는 USB 웹캠, 720p 이상, 30fps 권장 | 카메라 위치는 모니터 상단 중앙 |
| 핸드 트래킹 | MediaPipe Hands | 양손 21개 랜드마크, 좌/우 손 구분 | `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection` 샘플 씬과 스크립트 참고 |
| 입력 보조 | Unity Old Input | 운영자 리셋, 디버그 토글, 임시 성공 처리 | New Input System 사용하지 않음 |
| 공간 | 귀여운 단일 방 | 장난감 공방풍 1개 방 | 이동 가능 영역과 상호작용 영역을 명확히 구분 |
| 오브젝트 | 상호작용 프롭 5~7개 | 큐브, 퍼즐 상자, 별 램프, 쿠키 블록, 버튼, 로봇 장난감 | 집기/배치/파괴/인증 기능 우선 |
| UI | UGUI 우선 | 손 인식 상태, 이동 원, 중앙 타겟, 자막 가이드, 완료 게이지 | 1920x1080 기준 |
| 이미지 | 콘셉트 PNG, UI 참고 PNG | `Assets/Concept`에 저장 | SVG 사용하지 않음 |
| 오디오 | 효과음/환경음 | 잡기, 놓기, 성공, 실패, 파괴, 별 램프 점등 | WAV 또는 OGG |
| 텍스트 | 근접 자막/힌트 | 오브젝트 근처 접근 시 1~2문장 표시 | 한국어 우선 |

#### 에셋 메타데이터 기준
- 이미지:
  - 형식: PNG
  - 콘셉트 이미지 권장 해상도: 16:9, 1920x1080 기준
  - UI 아이콘 권장 해상도: 512x512 또는 1024x1024
  - 파일명 규칙: `cute_`, `ui_`, `prop_`, `concept_` 접두어 사용
  - 태그: `concept`, `cute_room`, `gesture`, `ui`, `object_interaction`
- 오디오:
  - 형식: WAV 또는 OGG
  - 샘플레이트: 44.1kHz 이상
  - 효과음 길이: 0.2~2초
  - 태그: `sfx_grab`, `sfx_place`, `sfx_break`, `sfx_success`, `ambience`
- 텍스트:
  - 한 문장 20자 내외를 우선한다.
  - 오브젝트 근처 안내, 성공 피드백, 손 인식 실패 안내로 구분한다.

### 2-2. 공간 테마 및 배치
- 전체 공간 콘셉트: 파스텔 장난감 공방처럼 귀엽게 꾸며진 한 개의 방
- 시각 톤:
  - 바닥은 민트색 러그와 밝은 나무 질감으로 구성한다.
  - 벽은 크림색, 가구는 코랄과 하늘색 포인트를 사용한다.
  - 상호작용 가능한 물체에는 시안색 외곽선과 작은 별 파티클을 적용한다.
- 컬러 팔레트:
  - 기본: `#FFF4DE` 크림, `#BFEBD8` 민트, `#8CC7F7` 스카이 블루
  - 포인트: `#FF8A7A` 코랄, `#FFD166` 웜 옐로
  - 인터랙션: `#20D9D2` 시안, `#FF5C7A` 오류 코랄
- 조명 방향:
  - 전체는 밝고 부드러운 확산광으로 유지한다.
  - 조작 가능한 오브젝트는 접근 시 외곽선, 바닥 원, 작은 글로우를 표시한다.
  - 완료 시 별 램프와 천장 조명이 따뜻한 노란색으로 켜진다.

| 배치 구역 | 목적 | 주요 오브젝트 | 인터랙션 |
| --- | --- | --- | --- |
| 시작 매트 | 손 인식과 이동 학습 | 발자국 모양 러그, 손 아이콘 패널 | 양손 인식, 왼손 이동 원 안내 |
| 중앙 테이블 | 기본 잡기 학습 | 색 큐브, 작은 컵, 퍼즐 상자 | 오른손 조준, 핀치 집기, 놓기 |
| 장난감 선반 | 오브젝트 탐색 | 로봇 장난감, 별 램프, 책 | 근접 자막, 손바닥 인증 |
| 쿠키 블록 코너 | 파괴 학습 | 깨지는 쿠키 블록, 말랑 버튼 | 오른손 주먹 파괴, 버튼 강타 |
| 출구/완료 벽 | 체험 완료 | 별 램프 게이지, 작은 문 | 완료 조건 표시, 클리어 연출 |

#### 플레이 동선
- 시작 매트에서 양손을 인식한다.
- 왼손으로 중앙 테이블까지 이동한다.
- 오른손으로 큐브를 집고 매트 위에 놓는다.
- 장난감 선반 쪽으로 이동해 별 램프 안내를 확인한다.
- 쿠키 블록 코너에서 주먹 파괴를 성공한다.
- 별 램프 앞에서 오른손 손바닥을 유지해 완료한다.

### 2-3. 스토리텔링 및 텍스트

#### 기본 안내 톤
- 문장은 어린 사용자도 이해할 수 있도록 짧고 다정하게 쓴다.
- 설명은 오브젝트 근처에서만 뜨게 해 화면을 복잡하게 만들지 않는다.
- 실패 안내는 꾸짖는 표현 대신 다시 시도할 행동을 알려준다.

#### 시작 안내
- `카메라가 양손을 볼 수 있게 양손을 올려 주세요.`
- `왼손은 이동, 오른손은 물건 조작이에요.`
- `왼손을 원 안에서 움직여 방을 둘러보세요.`

#### 오브젝트 근처 자막
| 오브젝트 | 표시 조건 | 자막 |
| --- | --- | --- |
| 색 큐브 | 1.5m 이내 접근 | `오른손으로 가리키고 핀치하면 큐브를 집어요.` |
| 배치 매트 | 큐브를 잡은 상태로 접근 | `핀치를 풀면 이 자리에 놓을 수 있어요.` |
| 퍼즐 상자 | 중앙 타겟이 상자에 닿음 | `손바닥을 잠깐 보여주면 상자가 반응해요.` |
| 쿠키 블록 | 1.5m 이내 접근 | `오른손 주먹을 쥐면 블록을 깨뜨려요.` |
| 말랑 버튼 | 중앙 타겟이 버튼에 닿음 | `주먹으로 버튼을 톡 눌러 보세요.` |
| 별 램프 | 조건 3개 이상 완료 | `오른손 손바닥을 2초 동안 보여주세요.` |

#### 성공/실패 피드백 문구
- 성공:
  - `좋아요! 물건을 집었어요.`
  - `딱 맞는 자리에 놓였어요.`
  - `반짝! 별 램프가 켜졌어요.`
- 실패/보정:
  - `손이 화면 밖으로 나갔어요. 가운데로 돌아와 주세요.`
  - `조금 더 가까이 가면 조작할 수 있어요.`
  - `핀치를 잠깐 더 유지해 주세요.`
- 완료:
  - `손짓 놀이방 체험 완료!`

### 2-4. 기술 사양
| 항목 | 내용 |
| --- | --- |
| 메인 지원 플랫폼 | Windows PC |
| 실행 방식 | Unity Standalone 빌드 |
| 웹 지원 | 프로토타입 후순위 검토 |
| 카메라 | Windows 호환 내장 카메라 또는 USB 웹캠 |
| 핸드 트래킹 | MediaPipe Hands, 양손 추적, 21개 랜드마크 |
| MediaPipe 참고 위치 | `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection` |
| 입력 정책 | 플레이어 조작은 핸드 트래킹, 운영자 보조는 Unity Old Input |
| 권장 해상도 | 1920x1080 |
| 권장 프레임 | 게임 렌더링 60fps, 카메라 입력 30fps 이상 |
| UI | 손 인식 상태, 이동 원, 중앙 타겟, 근접 자막, 완료 게이지 |
| 렌더링 | 밝은 3D 단일 방, 단순 PBR/Low-poly, 외곽선 피드백 |
| 네트워크 | 필수 아님 |
| 저장 | 프로토타입에서는 저장 없음 또는 로컬 옵션만 저장 |
| 접근성 | 입력 유지 시간 완화, 운영자 리셋, 자막 안내 유지 |

#### 구현 우선순위
- 1순위:
  - MediaPipe 양손 인식 연결
  - 왼손 원형 이동 영역
  - 오른손 중앙 타겟, 핀치 잡기, 주먹 파괴
  - 오브젝트 근접 자막
- 2순위:
  - 귀여운 단일 방 블록아웃
  - 주요 프롭 5~7개 배치
  - 성공/실패 이펙트와 효과음
- 3순위:
  - 콘셉트 이미지 기반 아트 패스
  - 별 램프 완료 연출
  - 힌트와 난이도 옵션

### 2-5. 콘셉트 이미지 / 무드보드
요청3 기준 콘셉트 이미지는 SVG가 아닌 PNG로 생성해 `Assets/Concept`에 저장한다.

| 파일명 | 용도 | 포함 요소 | 활용 가이드 |

#### 무드보드 방향
- 공간:
  - 한눈에 읽히는 작고 귀여운 방으로 구성한다.
  - 길 찾기보다 오브젝트와 손동작 반응이 잘 보이는 배치를 우선한다.
- UI:
  - 왼손 UI는 화면 왼쪽, 오른손 UI는 화면 오른쪽에 배치한다.
  - 중앙 타겟 십자는 항상 얇고 밝게 유지하되, 대상에 닿을 때만 강조한다.
  - 근접 자막은 화면 하단 중앙에 1~2줄로 표시한다.
- 폰트:
  - 한글 가독성이 좋은 둥근 산세리프체를 사용한다.
  - 제목은 굵게, 안내문은 중간 굵기로 구성한다.
- 조명:
  - 기본 조명은 따뜻하고 부드럽게 유지한다.
  - 상호작용 가능 상태는 시안색, 성공은 노란색, 실패는 코랄색으로 구분한다.

## 3. 프로토타입 기대 효과
- 한 개의 방만으로 손 추적 이동, 조준, 잡기, 놓기, 부수기, 인증의 핵심 재미를 빠르게 검증할 수 있다.
- 귀여운 공간과 짧은 자막 안내로 처음 체험하는 사용자도 조작 규칙을 쉽게 이해한다.
- MediaPipe Hands 샘플을 기반으로 구현 범위를 작게 유지하면서 Unity 3D 인터랙션으로 확장할 수 있다.
- 이후 방 확장, 오브젝트 추가, 전시형 운영 모드, 웹 빌드 검토로 자연스럽게 발전시킬 수 있다.
