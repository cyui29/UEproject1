# 핸드 트래킹 DJ 인터랙션 콘텐츠 기획서 v2

## 0. v2 반영 배경
- 기준 문서: `Assets/Design/콘텐츠기획.md`
- 실행 환경: PC Windows 단독 실행형 Unity 애플리케이션
- 입력 전제: 현재 프로젝트에 import된 MediaPipe 패키지 중 `Hand detection / Hands landmark`만 사용한다.
- 차별화 방향:
  - TouchDesigner식 노드 기반 영상 반응형 콘텐츠보다, Unity의 강점인 3D 공간, 물리, 카메라 연출, 런타임 상태 관리, 빌드 배포 안정성을 전면에 둔다.
  - 영상 필터 중심의 반응이 아니라, 사용자의 손이 3D 무대와 오디오 시스템을 직접 조종하는 퍼포먼스형 콘텐츠로 설계한다.
  - 프로토타입에서는 Hand detection 기반의 안정적인 제스처, Unity AudioMixer, Shader/VFX Graph, Timeline 또는 Cinemachine 연출을 우선 사용한다.

## 1. 시나리오 기획

### 1-1. 컨셉 및 기획 의도
- 프로젝트명: `Air DJ: Stage Mix`
- 핵심 메시지: 카메라 앞에서 손을 움직이는 순간, 사용자는 3D 무대의 DJ가 되어 음악, 빛, 공간 연출을 동시에 지휘한다.
- 기획 의도:
  - Windows PC 카메라와 MediaPipe Hands만으로 별도 장비 없는 핸드 트래킹 DJ 체험을 구현한다.
  - 손동작을 음악 파라미터뿐 아니라 Unity 3D 오브젝트, 조명, 카메라, 파티클, 포스트 프로세싱 연출에 연결한다.
  - TouchDesigner와 유사한 실시간 비주얼 반응을 넘어서, 게임 엔진의 장면 전환, 상태 머신, 3D 품질, 사용자 피드백 루프를 활용한다.
- 목적:
  - 짧은 체험 시간 안에 `내 손으로 무대가 바뀐다`는 성취감을 제공한다.
  - 이벤트, 전시, 키오스크 환경에서 반복 운영 가능한 Windows PC 기반 프로토타입 포맷을 확보한다.
  - 향후 곡 테마, 3D 스테이지 스킨, 점수/리플레이, 멀티 사용자 모드로 확장 가능한 구조를 만든다.

### 1-2. 타겟 정의
- 메인 타겟:
  - 음악 비전공자
  - 체험형 디지털 콘텐츠 이용자
  - 10대 후반~30대 일반 사용자
  - SNS 촬영과 공유를 즐기는 방문자
- 서브 타겟:
  - 가족 단위 체험객
  - 기업 행사 및 브랜드 체험 공간 방문자
  - Unity 기반 인터랙티브 콘텐츠를 처음 접하는 초심자
- 타겟 특성 분석:
  - 조작법을 오래 배우기보다 즉시 반응하는 화면과 음악을 기대한다.
  - 정확한 연주보다, 대충 움직여도 멋있게 보이는 보정과 연출을 선호한다.
  - 본인의 손 위치가 3D 무대의 조명, 파티클, 카메라 워크에 직접 반영될 때 몰입도가 높다.

### 1-3. 사용자 인터랙션
기본 원칙은 `한 손 = 연속 파라미터 제어`, `양손 = 무대 연출 변화`, `정지 = 상태 유지`, `큰 동작 = 하이라이트 이벤트`이다. 프로토타입에서는 손가락 세부 포즈보다 손바닥 위치, 손 중심점, 양손 거리, 손 이동 속도를 우선 사용한다.

| 손짓/행동 | MediaPipe 사용값 | 음악 변화 | Unity 화면 반응 |
| --- | --- | --- | --- |
| 손바닥 펼치기 | 손 감지 시작, wrist/palm 기준점 | 기본 비트와 배경 패드 재생 시작 | 3D DJ 콘솔 점등, 손 주변 링 이펙트 표시 |
| 오른손 위로 올리기 | 오른손 Y축 정규화 좌표 | 필터 오픈, 고역 강조 | 무대 상단 조명 밝기 증가, 파티클 상승 |
| 오른손 아래로 내리기 | 오른손 Y축 정규화 좌표 | 필터 클로즈, 저역 중심 | 조명 다운, 배경 파형 밀도 감소 |
| 왼손 위로 올리기 | 왼손 Y축 정규화 좌표 | 드럼/베이스/패드 레이어 추가 | 좌측 3D 레이어 블록 활성화 |
| 왼손 아래로 내리기 | 왼손 Y축 정규화 좌표 | 레이어 볼륨 감소 또는 뮤트 | 레이어 블록 비활성화, 색상 채도 감소 |
| 손을 좌우로 이동 | 손 중심 X축 | 크로스페이더 이동 | 3D 콘솔 중앙 미터가 좌우로 이동 |
| 양손 벌리기 | 양손 중심점 거리 | 리버브/공간감 확대, 빌드업 | 무대 벽면 확장, 카메라 FOV 소폭 증가 |
| 양손 모으기 | 양손 중심점 거리 | 리버브 축소, 사운드 집중 | 파티클이 중앙으로 수렴, 카메라가 콘솔 쪽으로 접근 |
| 손목 또는 손 중심 회전 | 손 랜드마크 방향 벡터 | 필터 스윕 또는 스크래치 느낌의 FX | 3D 다이얼 회전, 잔상 궤적 표시 |
| 빠른 손 튕김 | 손 중심 속도 임계값 | 킥, 스네어, FX 원샷 트리거 | 순간 플래시, 화면 펄스, 비트 링 확산 |
| 두 손 1초 정지 | 양손 속도 임계값 이하 | 현재 상태 홀드, 루프 잠금 | HOLD 텍스트, 손 외곽선 고정 연출 |
| 양손 크게 원형 이동 | 손 중심 궤적 변화 | 하이라이트 섹션 전환 | Timeline/Cinemachine 기반 무대 카메라 전환 |

- 난이도 설계:
  - 초급 모드: 손 Y축, X축, 양손 거리만 사용한다.
  - 일반 모드: 손 이동 속도, 정지 홀드, 큰 원형 이동을 추가한다.
  - 프로토타입 1차 목표: 초급 모드 완성 후 일반 모드 일부를 선택 구현한다.
- UX 설계 포인트:
  - 손 인식 실패 시 음악이 즉시 끊기지 않도록 0.5~1초 입력 보간 시간을 둔다.
  - 손 좌표는 스무딩과 데드존을 적용해 떨림을 줄인다.
  - 모든 음악 변화는 BPM과 스케일 안에서 보정해 불쾌한 사운드가 나오지 않게 한다.
  - 피드백 목표는 0.1~0.2초 이내이며, 오디오보다 화면 반응을 먼저 느낄 수 있게 손 주변 이펙트를 항상 표시한다.

### 2-1. 작품 라인업 및 스펙

#### 권장 메타데이터
- 이미지:
  - 파일 형식: PNG
  - 권장 해상도: 1920x1080 이상
  - 용도 태그: `stage`, `ui`, `guide`, `promo`
- 오디오:
  - 파일 형식: WAV 또는 고품질 OGG
  - BPM: 100 / 110 / 120 중 통일
  - 키: C minor 또는 A minor 중심
  - 길이: 4, 8, 16마디 루프 단위
- 3D/비주얼:
  - 반응형 무대 오브젝트: 콘솔, 라이트 타워, 파형 링, 레이어 블록
  - 이펙트 재료: 파티클 텍스처, 글로우 머티리얼, 라인 렌더러 또는 VFX Graph 프리셋

### 2-2. TouchDesigner 대비 Unity 차별화 포인트
| 차별화 축 | TouchDesigner에서 강한 부분 | Unity에서 기획상 더 밀어야 할 부분 |
| --- | --- | --- |
| 실시간 비주얼 | 노드 기반 영상 합성, 프로젝션, 센서 연동 | 게임 오브젝트 기반 3D 무대, 카메라 워크, 물리 반응 |
| 사용자 경험 | 즉흥적인 비주얼 패치와 실험 | 튜토리얼, 상태 전환, 실패 보정, 리플레이 등 완성형 UX |
| 배포 | 설치 환경과 패치 의존도 높음 | Windows 실행 파일로 패키징, 키오스크 운영에 유리 |
| 확장성 | 미디어 아트 설치에 강함 | 점수, 언락, 스킨, 세션 저장, 게임화 구조에 유리 |
| 품질 표현 | 영상 필터와 합성 중심 | 3D 라이팅, 포스트 프로세싱, VFX Graph, 캐릭터/무대 에셋 활용 |
| 운영 안정성 | 현장 튜닝에 유연 | 씬/프리팹/ScriptableObject 기반으로 반복 운영과 QA가 쉬움 |

#### Unity에서만 가능하거나 더 적합한 기획 요소
- 3D DJ 스테이지:
  - 손 위치에 따라 콘솔, 조명 타워, 배경 오브젝트가 실제 공간 안에서 움직인다.
  - 카메라가 드롭 순간에 줌 인/아웃하거나 회전해 게임 같은 하이라이트를 만든다.
- 게임화된 퍼포먼스 피드백:
  - 정확한 리듬 입력보다 에너지, 다양성, 연속 제스처를 점수화한다.
  - 체험 종료 후 `Stage Rank`, `Best Gesture`, `Peak Energy` 같은 결과 화면을 제공한다.
- 물리 기반 오브젝트 반응:
  - 손 튕김으로 3D 비트 오브젝트를 밀어내거나, 양손 거리로 파형 링의 반지름을 바꾼다.
  - 파티클과 라이트가 오디오 분석값뿐 아니라 손 추적 상태에 함께 반응한다.
- 운영자 친화 구조:
  - 키보드로 리셋, 튜토리얼 스킵, 곡 변경, 볼륨 조절을 제공한다.
  - 씬 전환과 상태 관리를 코드로 명확히 나눠 전시 운영 중 예외 대응이 쉽다.
- 고품질 빌드 결과물:
  - Windows PC에서 고정 해상도와 성능 목표를 맞춘 실행 파일을 제공한다.
  - TouchDesigner 프로젝트 파일이 아니라 관람객용 앱처럼 완성된 시작, 플레이, 결과 흐름을 제공한다.

### 2-3. 공간 테마 및 배치
- 공간 콘셉트: `미래형 사운드 랩 + 게임 엔진 기반 인터랙티브 스테이지`
- 공간 무드:
  - 메인 컬러: 딥 차콜, 틸, 시안
  - 포인트 컬러: 웜 오렌지, 코랄
  - 조명: 사용자의 손 위치, 음악 강약, 섹션 상태에 반응하는 무대 조명
- 권장 섹션 구성:

| 섹션 | 역할 | 배치 요소 | 색상/조명 모드 |
| --- | --- | --- | --- |
| Entrance | 대기 및 유입 | 타이틀 월, 체험 방법 요약, 대기선 | 어두운 배경 + 은은한 틸 라이트 |
| Tutorial Zone | 제스처 학습 | 손 아이콘, 모션 예시, 카메라 위치 안내 | 밝은 시안 계열, 명확한 정보 조명 |
| Play Zone | 메인 체험 | 대형 디스플레이, 카메라, 스탠딩 포지션, 메인 오디오 | 틸/오렌지 대비, 반응형 조명 강화 |
| Result Zone | 결과 감상 | 랭크, 베스트 제스처, 캡처 화면 | 코랄 포인트, 리플레이 무드 |

- 배치 원칙:
  - 카메라는 디스플레이 상단 중앙에 위치해 사용자의 양손이 안정적으로 잡히게 한다.
  - 사용자의 스탠딩 위치는 화면과 1.8m~2.5m 거리로 표시한다.
  - 조명은 손 인식에 방해되지 않도록 얼굴과 손 영역을 과도하게 어둡게 만들지 않는다.
  - PC Windows 운영 환경을 기준으로 키보드/마우스가 운영자 위치에서 접근 가능해야 한다.

### 2-4. 스토리텔링 및 텍스트
#### 작품 설명
`Air DJ: Unity Stage Mix`는 손짓으로 음악과 3D 무대를 동시에 움직이는 인터랙티브 퍼포먼스입니다. 사용자는 전문 장비 없이 카메라 앞에 서서 손을 올리고, 벌리고, 튕기는 동작만으로 사운드 레이어와 필터, 공간감, 조명, 파티클, 카메라 연출을 조종합니다. Unity 기반의 3D 무대는 사용자의 손 위치를 실시간으로 반영하며, 체험 마지막에는 나만의 퍼포먼스 결과를 게임처럼 확인할 수 있습니다.

#### 메인 텍스트
- 메인 카피: `손끝으로 무대를 믹스하다`
- 서브 카피: `카메라 앞에 서는 순간, 당신의 제스처가 음악과 빛의 무대가 됩니다.`

#### 팝업 안내문
- 대기 화면:
  - `화면 중앙에 서서 두 손을 들어주세요.`
  - `손이 인식되면 무대와 사운드가 시작됩니다.`
- 플레이 중:
  - `오른손을 위로 올리면 사운드가 밝아집니다.`
  - `왼손을 올리면 새로운 레이어가 켜집니다.`
  - `양손을 벌리면 무대 공간이 확장됩니다.`
  - `손을 빠르게 튕기면 비트 이펙트가 터집니다.`
- 결과 화면:
  - `오늘의 무대 에너지`
  - `베스트 제스처`
  - `다시 플레이`
- 오류/예외:
  - `손이 화면에서 벗어났습니다. 중앙으로 다시 이동해주세요.`
  - `주변 조명이 너무 어두우면 인식률이 낮아질 수 있습니다.`
  - `카메라 연결을 확인해주세요.`

### 2-5. 기술 사양
| 항목 | 내용 |
| --- | --- |
| 메인 지원 플랫폼 | PC Windows |
| 실행 방식 | 단독 실행형 Unity 애플리케이션 |
| 입력 장치 | 웹캠, 키보드/마우스 운영자 보조 |
| 손 추적 범위 | MediaPipe Hands 기반 양손 감지, 21개 랜드마크 |
| 입력 시스템 | Unity Old Input 기준 |
| 권장 해상도 | Full HD(1920x1080) 이상 |
| 권장 프레임 | 60fps |
| 카메라 권장값 | 1080p, 30fps 이상, 정면 고정 |
| 오디오 출력 | 스테레오 스피커 또는 현장 음향 시스템 |
| 네트워크 | 필수 아님, 오프라인 실행 가능 구조 권장 |
| 확장 가능 플랫폼 | 키오스크형 Windows 디바이스, 대형 디스플레이 연동 시스템 |

### 2-6. 프로토타입 MVP 범위
- 반드시 구현:
  - 손 감지 시작/종료 상태
  - 오른손 Y축 기반 필터 제어
  - 왼손 Y축 기반 레이어 볼륨 제어
  - 양손 거리 기반 리버브 또는 공간감 제어
  - 손 주변 링/파티클 피드백
  - 키보드 리셋과 운영자 보조 입력
- 가능하면 구현:
  - 빠른 손 튕김 원샷 FX
  - 하이라이트 순간 카메라 전환
  - 결과 화면의 간단한 에너지 랭크
- 제외 또는 후순위:
  - 손가락별 복잡한 사인 언어형 제스처
  - 네트워크 공유 기능
  - 고급 음악 생성 AI
  - 멀티 사용자 대전

## 기대 효과
- TouchDesigner식 실시간 영상 반응과 겹치지 않고, Unity의 3D 무대/게임화/배포 안정성을 앞세운 체험 콘셉트를 확보한다.
- MediaPipe Hands만으로도 구현 가능한 좌표 기반 제스처를 중심으로 프로토타입 리스크를 낮춘다.
- PC Windows 단독 실행형 콘텐츠로 전시 및 이벤트 운영에 적합한 구조를 갖춘다.
- 향후 스킨, 점수, 리플레이, 곡 테마 확장으로 콘텐츠 수명을 늘릴 수 있다.
