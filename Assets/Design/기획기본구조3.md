# Hand Play Room 시스템 기획 기본 구조

## 1. 시스템 기획서

### 프로젝트명
`Hand Play Room`

### 기준 콘텐츠 기획서
- `Assets/Design/콘텐츠기획3.md`

### 콘텐츠 개요
`Hand Play Room`은 Windows PC 웹캠과 MediaPipe Hands를 활용해 사용자의 양손을 추적하고, 귀엽게 꾸며진 단일 3D 방 안에서 이동과 오브젝트 상호작용을 체험하는 Unity 프로토타입 콘텐츠다.

사용자는 왼손으로 방 안을 이동하거나 시선을 조절하고, 오른손으로 중앙 타겟을 통해 오브젝트를 조준한다. 오른손 핀치로 큐브나 장난감 오브젝트를 집고 놓으며, 오른손 주먹으로 쿠키 블록이나 버튼을 강하게 상호작용한다. 오브젝트 근처에 접근하면 짧은 자막 가이드가 표시되어 사용자가 별도 설명 없이도 다음 행동을 이해할 수 있다.

프로토타입은 복잡한 스토리나 여러 방을 만드는 대신, 한 개의 작은 장난감 공방 안에서 손 인식, 이동, 조준, 집기, 놓기, 부수기, 인증, 클리어 피드백의 핵심 루프를 빠르게 검증하는 것을 목표로 한다.

### 프로토타입 목표
- 3~5분 안에 체험 가능한 단일 방 손동작 인터랙션 루프를 완성한다.
- 왼손 이동과 오른손 상호작용의 역할 분리를 명확하게 체험시킨다.
- MediaPipe Hands 기반 양손 인식 결과를 Unity 3D 공간 조작으로 연결한다.
- 오브젝트 근처 자막, 외곽선, 파티클, 완료 게이지를 통해 즉각적인 피드백을 제공한다.
- 상호작용 오브젝트는 5~7개 범위로 제한하고, 구현과 테스트 범위를 작게 유지한다.

### 제외 또는 후순위 범위
- 멀티플레이
- 네트워크 연동
- 장기 저장 시스템
- 복수 방 확장
- 복잡한 인벤토리 또는 퀘스트 시스템
- Unity New Input System 기반 입력 구조

## 2. 핵심 컨셉

### 한 줄 정의
웹캠 앞의 양손 움직임만으로 귀여운 3D 방을 이동하고 오브젝트를 직접 조작하는 핸드 트래킹 인터랙션 콘텐츠.

### 지향 경험
- 사용자는 컨트롤러 없이 자신의 손이 곧 조작 도구가 되는 느낌을 받아야 한다.
- 왼손은 이동, 오른손은 상호작용이라는 규칙이 첫 체험 안에서 바로 이해되어야 한다.
- 방 안 오브젝트는 가까이 다가가면 말을 걸듯 짧은 안내를 제공해야 한다.
- 집기, 놓기, 부수기, 인증 같은 행동은 화면 피드백과 효과음으로 즉시 확인되어야 한다.
- 실패해도 바로 다시 시도할 수 있는 가벼운 놀이방 체험이어야 한다.

### 사용자 가치
- 손동작 기반 인터랙션을 짧은 시간 안에 직관적으로 체험할 수 있다.
- 복잡한 조작 학습 없이 화면 자막과 오브젝트 반응만 보고 다음 행동을 알 수 있다.
- 자신의 손이 3D 공간 안의 물건에 직접 영향을 준다는 몰입감을 얻을 수 있다.
- 전시, 수업, 프로토타입 발표에서 핵심 기술과 콘텐츠 재미를 빠르게 보여줄 수 있다.

### 차별 포인트
- 양손의 역할을 명확히 나눈다.
  - 왼손: 이동과 시선
  - 오른손: 조준과 오브젝트 조작
- 단일 방 구조로 범위를 줄이면서도 완결된 체험 흐름을 제공한다.
- 귀여운 파스텔 장난감 공방 톤으로 손 추적 기술 데모의 딱딱함을 낮춘다.
- 오브젝트 근처 자막 가이드로 별도 튜토리얼 화면 없이 조작법을 자연스럽게 전달한다.

## 3. 핵심 메커니즘

### 3-1. 손 인식 시작
#### 목적
체험 시작 전에 카메라가 사용자의 손을 안정적으로 인식하는지 확인한다.

#### 작동 방식
- 사용자가 시작 매트 영역에서 양손을 화면 중앙에 보여준다.
- MediaPipe Hands가 왼손과 오른손을 구분해 감지한다.
- 양손 또는 최소 한 손이 일정 시간 이상 안정적으로 인식되면 체험 대기 상태에서 플레이 상태로 전환한다.

#### 피드백
- 왼손 인식 완료
- 오른손 인식 완료
- 손이 화면 밖으로 나가면 중앙으로 돌아오라는 안내 표시

### 3-2. 왼손 이동/시선 제어
#### 목적
왼손을 이동 전용 입력으로 사용해 단일 방 안을 탐색한다.

#### 작동 방식
- 화면 왼쪽 또는 하단에 원형 이동 영역을 표시한다.
- 왼손 손바닥 중심 좌표를 기준으로 원 안의 상대 위치를 계산한다.
- 손 중심점이 원의 왼쪽/오른쪽으로 이동하면 좌우 이동 또는 회전을 수행한다.
- 손 중심점이 위/아래로 이동하면 전진/후진 또는 시선 상하 조절을 수행한다.
- 왼손 주먹 판정이 들어오면 이동을 정지한다.

#### 보정 원칙
- 손 중심 좌표에 짧은 스무딩을 적용한다.
- 손이 화면 밖으로 나가면 이동 입력을 즉시 정지한다.
- 이동 속도는 프로토타입에서 멀미가 나지 않도록 낮게 시작한다.

### 3-3. 오른손 타겟 조준
#### 목적
오른손을 오브젝트 선택과 상호작용의 기준 입력으로 사용한다.

#### 작동 방식
- 화면 중앙에 얇은 타겟 십자를 표시한다.
- 오른손 검지 방향 또는 손 중심 방향을 기준으로 조준 레이를 계산한다.
- 조준 레이가 상호작용 가능 오브젝트에 닿으면 오브젝트 외곽선과 가이드 자막을 표시한다.

#### 피드백
- 조준 가능한 오브젝트에는 시안색 외곽선 표시
- 조준 중인 오브젝트 이름 또는 짧은 행동 안내 표시
- 조준 해제 시 외곽선과 안내 제거

### 3-4. 핀치 기반 오브젝트 집기/놓기
#### 목적
오른손 엄지-검지 핀치로 오브젝트를 집고 원하는 위치에 놓는다.

#### 작동 방식
- 오른손 엄지 끝과 검지 끝 사이의 거리가 임계값 이하가 되면 핀치로 판정한다.
- 조준 중인 오브젝트가 잡기 가능 상태라면 오브젝트를 플레이어 앞 약 1m 프리뷰 위치로 이동시킨다.
- 핀치를 유지하는 동안 오브젝트는 조준점 앞에 따라온다.
- 핀치를 해제하면 현재 위치 또는 배치 가능 매트에 오브젝트를 놓는다.

#### 실패/안전 처리
- 잡는 중 손 인식이 끊기면 오브젝트를 현재 위치에 내려놓는다.
- 오브젝트가 방 밖이나 바닥 아래로 떨어지면 시작 위치로 자동 복귀한다.
- 배치 불가 지점에는 오류 색상 피드백을 짧게 표시한다.

### 3-5. 주먹 기반 오브젝트 파괴/버튼 강타
#### 목적
오른손 주먹으로 강한 상호작용을 표현한다.

#### 작동 방식
- 오른손 손가락 끝들이 손바닥 방향으로 접힌 상태를 주먹으로 판정한다.
- 조준 중인 오브젝트가 파괴 가능 상태라면 깨짐 상태로 전환한다.
- 버튼형 오브젝트라면 강타 입력으로 처리한다.

#### 피드백
- 쿠키 블록 또는 깨지는 오브젝트에 파편/흔들림 효과 표시
- 버튼 강타 시 짧은 눌림 애니메이션과 효과음 재생
- 성공 시 별 파티클 또는 노란색 완료 피드백 표시

### 3-6. 오브젝트 근접 자막 가이드
#### 목적
사용자가 방 안 오브젝트 근처에 접근했을 때 다음 행동을 자연스럽게 알게 한다.

#### 작동 방식
- 플레이어와 상호작용 오브젝트 사이의 거리를 계산한다.
- 일정 거리 이내에 들어오면 하단 중앙에 1~2줄 안내 자막을 표시한다.
- 현재 조준/잡기/완료 상태에 따라 안내 문구를 바꾼다.

#### 예시 문구
- `오른손으로 가리키고 핀치하면 큐브를 집어요.`
- `핀치를 풀면 이 자리에 놓을 수 있어요.`
- `오른손 주먹을 쥐면 블록을 깨뜨려요.`
- `오른손 손바닥을 2초 동안 보여주세요.`

### 3-7. 별 램프 완료/클리어 흐름
#### 목적
사용자가 주요 상호작용을 충분히 체험했을 때 명확한 완료감을 제공한다.

#### 작동 방식
- 큐브 집기, 큐브 배치, 쿠키 블록 부수기, 버튼 강타, 별 램프 인증 같은 완료 항목을 기록한다.
- 최소 3개 이상의 주요 상호작용이 완료되면 별 램프를 활성화한다.
- 별 램프 앞에서 오른손 손바닥을 2초 유지하면 클리어 상태로 전환한다.

#### 피드백
- 별 램프 점등
- 완료 게이지 상승
- 클리어 자막 표시
- 짧은 성공 효과음과 조명 변화

### 3-8. 운영자 리셋/디버그 보조
#### 목적
전시나 개발 중 손 인식 실패, 오브젝트 낙하, 진행 막힘을 빠르게 복구한다.

#### 작동 방식
- Unity Old Input 기반 키보드 입력으로 리셋, 진행 단계 강제 완료, 디버그 표시 토글을 제공한다.
- 운영자 입력은 플레이어 손동작 입력과 분리한다.
- New Input System은 사용하지 않는다.

#### 후보 입력
- `R`: 방 상태 리셋
- `H`: 힌트 표시 토글
- `D`: 디버그 좌표/제스처 상태 표시
- `C`: 클리어 조건 강제 충족

## 4. 기술 명세

프로토타입 단계에서는 Controller 수를 6개로 제한한다. 손 추적과 제스처 판별, 진행 조건과 HUD 피드백처럼 강하게 붙어 움직이는 책임은 하나의 시스템으로 묶어 구현과 Inspector 연결 부담을 줄인다.

### 4-1. 경험 흐름 및 씬 제어 시스템
- 후보 클래스: `UE_HandPlayroomFlowController`
- 책임:
  - 시작 대기, 손 인식 대기, 플레이 중, 클리어, 리셋 상태 관리
  - 씬 내 주요 시스템 초기화 순서 제어
  - 클리어 또는 리셋 시 전체 상태 복구 요청
- 주요 입력:
  - 손 인식 완료
  - 클리어 조건 충족
  - 운영자 리셋 입력
- 주요 출력:
  - 현재 경험 상태
  - UI 표시 상태
  - 방 진행 상태 초기화 명령
- 참고 위치:
  - `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection`

### 4-2. 웹캠 손 추적 및 제스처 판별 시스템
- 후보 클래스: `UE_HandTrackingGestureController`
- 책임:
  - 웹캠 입력 준비
  - MediaPipe Hands 샘플 연동 지점 관리
  - 왼손/오른손 21개 랜드마크 결과 수신
  - 좌표 정규화, 좌우 손 구분, 신뢰도 상태 관리
  - 손바닥, 주먹, 핀치, 손바닥 유지, 검지 조준 상태 판별
  - 제스처 임계값과 최소 유지 시간 관리
- 참고 위치:
  - `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection`
- 주요 입력:
  - 웹캠 프레임
  - MediaPipe Hands 추론 결과
- 주요 출력:
  - 안정화된 왼손 상태
  - 안정화된 오른손 상태
  - 손 인식/미인식 이벤트
  - 왼손 이동 벡터
  - 오른손 핀치 상태
  - 오른손 주먹 상태
  - 오른손 손바닥 인증 상태

### 4-3. 플레이어 이동 및 시선 제어 시스템
- 후보 클래스: `UE_HandPlayerController`
- 책임:
  - 왼손 이동 벡터를 플레이어 이동 또는 회전으로 변환
  - 시선 상하 조절 처리
  - 주먹 정지 입력 반영
  - 이동 가능 영역 제한
- 주요 입력:
  - 왼손 손바닥 중심 좌표
  - 왼손 주먹 상태
- 주요 출력:
  - 플레이어 위치
  - 카메라 또는 시선 방향
  - 이동 상태 피드백

### 4-4. 오브젝트 상호작용 시스템
- 후보 클래스: `UE_HandObjectInteractionController`
- 책임:
  - 중앙 타겟 조준 레이 처리
  - 상호작용 가능 오브젝트 감지
  - 핀치 잡기/놓기 처리
  - 주먹 파괴/버튼 강타 처리
  - 오브젝트 안전 복귀 처리
- 주요 입력:
  - 오른손 조준 상태
  - 오른손 핀치 상태
  - 오른손 주먹 상태
  - 플레이어 위치
- 주요 출력:
  - 현재 조준 오브젝트
  - 잡힌 오브젝트 상태
  - 오브젝트 완료 이벤트

### 4-5. 방 진행 및 피드백 시스템
- 후보 클래스: `UE_HandPlayroomProgressFeedbackController`
- 책임:
  - 주요 상호작용 완료 여부 기록
  - 별 램프 활성화 조건 판단
  - 클리어 조건 판단
  - 손 인식 상태 UI 표시
  - 왼손 이동 원 UI 표시
  - 중앙 타겟 십자 표시

## 5. 시스템 간 흐름

### 기본 플레이 흐름
1. `UE_HandPlayroomFlowController`가 시작 대기 상태로 진입한다.
2. `UE_HandTrackingGestureController`가 웹캠과 MediaPipe Hands 결과를 준비하고 왼손/오른손 제스처 상태를 판별한다.
4. 왼손 입력은 `UE_HandPlayerController`로 전달되어 이동과 시선 조절에 사용된다.
5. 오른손 입력은 `UE_HandObjectInteractionController`로 전달되어 조준, 집기, 놓기, 파괴에 사용된다.
6. 완료 이벤트와 근접 오브젝트 상태는 `UE_HandPlayroomProgressFeedbackController`가 진행률, 자막, HUD로 표시한다.
7. 완료 조건이 충족되면 별 램프 인증이 활성화되고, 클리어 상태로 전환된다.

### 리셋 흐름
1. 운영자가 Old Input 리셋 키를 입력한다.
2. `UE_HandPlayroomOperatorController`가 리셋 명령을 보낸다.
3. `UE_HandPlayroomFlowController`가 전체 시스템에 초기화 요청을 보낸다.
4. 플레이어 위치, 오브젝트 위치, 진행 상태, HUD 상태가 시작 상태로 복구된다.

## 6. 검증 기준

### 문서 검증
- `Assets/Design/콘텐츠기획3.md`의 단일 방 방향과 충돌하지 않는다.
- 시스템 항목이 이후 Task 문서로 분리 가능한 수준으로 명확하다.
- 모든 신규 public 클래스 후보가 `UE_` 접두어를 사용한다.
- Unity New Input System을 전제로 하지 않는다.

### 프로토타입 검증
- 시작 매트에서 손 인식 대기 상태가 확인된다.
- 왼손 이동으로 플레이어가 방 안에서 움직인다.
- 오른손 조준으로 상호작용 가능 오브젝트가 강조된다.
- 오른손 핀치로 오브젝트를 집고 놓을 수 있다.
- 오른손 주먹으로 파괴 또는 버튼 강타가 동작한다.
- 오브젝트 근처에서 자막 가이드가 표시된다.
- 주요 상호작용 완료 후 별 램프 클리어 흐름이 동작한다.
- 운영자 리셋으로 시작 상태를 복구할 수 있다.
