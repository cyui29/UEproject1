# 단계별 수행 절차

## 목적
- Unity에서 바로 보이는 결과를 먼저 만든 뒤 입력, 제스처, 운영자 제어를 연결한다.
- 기준 씬은 `Assets/Scenes/HandPlayroom.unity`로 한다.
- 기준 기획 문서는 `Assets/Design/콘텐츠기획.md`로 한다.
- `Assets/Tasks`의 6개 Task는 `1 task = 1 class` 원칙을 유지한다.
- Play Mode를 실제로 돌리지 않은 항목은 체크리스트에서 미검증으로 남긴다.

## `단계 실행`의 정의
사용자가 `단계 실행`이라고 요청하면 해당 단계는 단순 스크립트 생성만 의미하지 않는다. 다음 작업을 함께 완료 대상으로 본다.

1. 필요한 `UE_` 접두어 C# 스크립트 생성 또는 갱신
2. `HandPlayroom.unity`에 GameObject 배치
3. 필요한 기본 UI, 3D 오브젝트, Collider, Light, Camera 보조 오브젝트 배치
4. 컴포넌트 연결과 기본 Inspector 값 설정
5. Unity Console 컴파일 오류 확인
6. 씬 저장
7. 대응 Task 체크리스트 갱신

## 공통 진행 규칙
- 새 public 클래스명은 반드시 `UE_` 접두어를 사용한다.
- 입력 관련 구현은 Unity New Input System이 아닌 Old Input 기준으로 처리한다.
- 각 단계는 가능하면 서브에이전트에 독립 Task로 맡긴다.
- 서브에이전트에는 대상 Task 문서와 대응 체크리스트를 함께 전달한다.
- 다른 단계의 클래스는 필요한 참조 슬롯이나 public 연결 지점까지만 만든다.
- 실제 Play Mode 확인 전에는 동작 확인 항목을 `[x]`로 바꾸지 않는다.

## 실행 순서 요약
1. 씬 기준선과 UI/HUD 자리 만들기
2. `UE_HandPlayroomFlowController`로 대기/플레이/클리어/리셋 흐름 연결
3. 플레이 공간과 상호작용 대상 블록아웃
4. `UE_HandPlayroomProgressFeedbackController`로 화면 피드백 연결
5. `UE_HandTrackingGestureController`로 Mock 손 입력과 제스처 상태 연결
6. MediaPipe Hands 샘플 결과를 게임용 손 입력으로 변환하는 연결 단계 수행
7. `UE_HandPlayerController`로 왼손 이동 연결
8. `UE_HandObjectInteractionController`로 오른손 조준/집기/파괴 연결
9. `UE_HandPlayroomOperatorController`로 Old Input 운영자 키 연결
10. 전체 연결 검토 후 실제 확인한 항목만 체크리스트에 반영

## 0단계. 씬 기준선과 UI 뼈대 만들기

### 목표
- 기능 구현 전에도 손 인식 대기, 목표 안내, 진행 상태가 화면에서 보이는 상태를 만든다.
- 이후 Task 05 피드백 시스템이 연결할 UI 참조 대상을 먼저 준비한다.

### Unity 실습 절차
1. `Assets/Scenes/HandPlayroom.unity`를 연다.
2. 씬 루트에 빈 GameObject `UE_HandPlayroomRoot`를 만든다.
3. `UE_HandPlayroomRoot` 아래에 `Systems`, `Player`, `Rooms`, `Interactables`, `UI` 빈 GameObject를 만든다.
4. `Main Camera`를 `Player` 아래로 옮기거나, 별도 Camera Rig를 만들고 `Player` 기준 전방을 보도록 배치한다.
5. `UI` 아래에 `UE_MainCanvas`를 만들고 `Screen Space - Overlay`로 설정한다.
6. `EventSystem`이 없다면 생성한다.
7. `UE_MainCanvas` 아래에 `HandStatusPanel`, `ObjectivePanel`, `ProgressPanel`, `CenterReticle`, `PromptPanel`, `DebugPanel`을 만든다.
8. `HandStatusPanel`에는 왼손/오른손 인식 상태 텍스트 자리를 만든다.
9. `ObjectivePanel`에는 현재 방 목표 텍스트 자리를 만든다.
10. `ProgressPanel`에는 완료 항목 3개 이상을 표현할 게이지 또는 체크 아이콘 자리를 만든다.
11. `CenterReticle`에는 중앙 조준점 이미지를 둔다.
12. `PromptPanel`에는 근접 오브젝트 안내와 성공/실패 문구 텍스트 자리를 만든다.
13. `DebugPanel`은 비활성화 상태로 두고 손 좌표/제스처 상태 텍스트 자리를 만든다.
14. 씬을 저장한다.

### 완료 후 체크
- Hierarchy에서 `UE_HandPlayroomRoot`와 하위 루트 5개가 보인다.
- UI 패널들이 이름으로 구분되어 있고 Task 05에서 참조할 수 있다.

## 4.5단계. MediaPipe Hands 실제 연결 단계

### 목표
- `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection` 샘플의 웹캠/MediaPipe 실행 결과를 Hand Playroom 입력 시스템에 연결한다.
- 샘플 Runner를 게임 로직에 직접 섞지 않고, MediaPipe 결과를 게임용 요약 상태로 변환하는 어댑터를 둔다.
- `UE_HandTrackingGestureController` 외부에는 MediaPipe 원본 랜드마크를 직접 노출하지 않는다.

### 참고 샘플 파일
- `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection/Hand Landmark Detection.unity`
  - 샘플 씬이다. Hand Playroom 씬으로 직접 대체하지 않고, Runner와 Annotation 배치 구조를 참고한다.
- `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection/HandLandmarkerRunner.cs`
  - 웹캠 프레임을 MediaPipe `HandLandmarker`에 넣고 `HandLandmarkerResult`를 받는 핵심 실행 루프다.
- `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection/HandLandmarkDetectionConfig.cs`
  - 모델 경로, 실행 모드, 손 개수, confidence 기본값을 정의한다.
- `Assets/MediaPipeUnity/Samples/Scenes/Hand Landmark Detection/HandLandmarkDetectionConfigWindow.cs`
  - 샘플 설정 UI다. 프로토타입 게임 연결에는 필수로 사용하지 않는다.

### 연결 구조
```text
WebCam
→ HandLandmarkerRunner
→ HandLandmarkerResult
→ UE_MediaPipeHandTrackingAdapter
→ UE_HandTrackingGestureController.SubmitMediaPipeHandState(...)
→ Player / Interaction / HUD
```

### Unity 실습 절차
1. `Assets/Scripts/UE_MediaPipeHandTrackingAdapter.cs`를 생성한다.
2. `UE_MediaPipeHandTrackingAdapter`는 `HandLandmarkerResult`를 받아 손별 요약값만 계산한다.
3. 어댑터는 `result.handedness`, `result.handLandmarks`만 읽고, 원본 리스트를 다른 시스템으로 넘기지 않는다.
4. `Systems` 아래에 GameObject `UE_MediaPipeHandTrackingAdapter`를 만들고 생성한 컴포넌트를 붙인다.
5. 어댑터의 `UE_HandTrackingGestureController` 참조를 Inspector에 연결한다.
6. `HandLandmarkerRunner.cs`에 어댑터 참조 슬롯을 추가한다.
7. `HandLandmarkerRunner.OnHandLandmarkDetectionOutput(...)`에서 Annotation 표시 후 어댑터의 결과 처리 메서드를 호출한다.
8. `IMAGE` 또는 `VIDEO` 모드를 사용할 경우 `DrawNow(result)`가 호출되는 분기에도 같은 어댑터 호출을 추가한다.
9. Config 기본값은 우선 `RunningMode.LIVE_STREAM`, `NumHands = 2`, confidence `0.5` 기준으로 둔다.
10. 카메라 미러링 때문에 좌우 손이 반대로 들어오면 어댑터에 `swapHands` 옵션을 두고 Inspector에서 조정한다.
11. Unity Console 컴파일 오류를 확인한다.
12. 씬을 저장한다.

### 어댑터 변환 기준
- 손 구분은 `result.handedness[i].categories[0].categoryName`의 `"Left"` 또는 `"Right"`를 기준으로 한다.
- 손 중심 좌표는 21개 normalized landmark의 평균 `x`, `y`로 계산한다.
- 핀치는 엄지 끝 `4`와 검지 끝 `8`의 거리가 임계값 이하인지로 판단한다.
- 주먹은 손목 `0` 기준 손가락 끝 `8`, `12`, `16`, `20`이 가까운지로 판단한다.
- 손바닥 상태는 손가락 끝들이 충분히 펼쳐진 상태를 기준으로 계산하고, 유지 시간 판정은 `UE_HandTrackingGestureController`의 hold 설정에 맡긴다.
- 기본 landmark 인덱스는 `0 wrist`, `4 thumb tip`, `8 index tip`, `12 middle tip`, `16 ring tip`, `20 pinky tip`으로 본다.

### Runner 수정 기준
`HandLandmarkerRunner.cs`의 Live Stream 콜백은 다음 위치를 연결 지점으로 본다.

```csharp
private void OnHandLandmarkDetectionOutput(HandLandmarkerResult result, Image image, long timestamp)
{
  _handLandmarkerResultAnnotationController.DrawLater(result);
  handTrackingAdapter?.Apply(result);
}
```

### 완료 후 체크
- `UE_MediaPipeHandTrackingAdapter` public 클래스가 생성되어 있다.
- MediaPipe 결과가 `UE_HandTrackingGestureController.SubmitMediaPipeHandState(...)`로 전달된다.
- 샘플 Annotation 표시는 유지하되, 게임 로직은 어댑터를 통해서만 연결된다.
- 원본 MediaPipe landmark 리스트를 Player, Interaction, HUD 시스템에 직접 넘기지 않는다.
- 좌우 손 반전 보정 옵션이 있다.
- 실제 웹캠/MediaPipe 동작은 Play Mode에서 확인한 뒤에만 검증 완료로 표시한다.
