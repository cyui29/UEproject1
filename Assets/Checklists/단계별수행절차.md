# 단계별 수행 절차

## 목적
- Unity에서 바로 보이는 UI를 먼저 만들고, 이후 입력/제스처/운영자 제어를 붙인다.
- `Assets/Tasks`의 6개 Task는 `1 task = 1 class` 원칙을 유지한다.
- Task 문서 번호는 클래스 문서 식별 번호로 유지하되, 실제 실행 순서는 UI 우선 흐름을 따른다.
- 단계 실행은 스크립트 생성뿐 아니라 `SampleScene`에 GameObject 배치, 컴포넌트 연결, 기본 Inspector 설정까지 포함한다.
- 각 Task 완료 후 `Assets/Checklists/Tasks`의 대응 체크리스트를 갱신한다.
- Play Mode를 실제로 돌리지 않은 항목은 반드시 `Play Mode 미검증`으로 남긴다.

## 공통 진행 규칙
- 각 Task는 가능하면 서브에이전트로 독립 실행한다.
- 서브에이전트에게는 대상 Task 문서 1개와 대응 체크리스트 1개만 기준으로 전달한다.
- 새 C# public 클래스 이름은 반드시 `UE_` 접두어를 유지한다.
- 입력 처리는 Unity New Input System이 아닌 Old Input 기준으로 작성한다.
- 씬 작업은 `Assets/Scenes/SampleScene.unity`를 기준으로 진행한다.
- 구현 후에는 Unity Console 컴파일 오류가 없는지 확인한다.
- 체크리스트는 실제로 확인한 항목만 `[x]`로 바꾼다.

## 0단계. 씬과 UI 뼈대 먼저 만들기

### 목표
- 사용자가 Unity Hierarchy에서 바로 확인할 수 있는 기본 화면 구조를 만든다.
- 아직 기능이 없어도 타이틀, 음악 선택, 플레이 HUD 영역이 씬에 보여야 한다.

### Unity 실습 절차
1. `Assets/Scenes/SampleScene.unity`를 연다.
2. 씬 루트에 빈 GameObject를 만들고 이름을 `UE_AppRoot`로 지정한다.
3. `Canvas`를 만들고 이름을 `UE_MainCanvas`로 지정한다.
4. `EventSystem`이 없다면 생성한다.
5. `UE_MainCanvas` 아래에 `TitlePanel`, `MusicSelectionPanel`, `PlayHudPanel`을 만든다.
6. `TitlePanel`에는 프로젝트명과 시작 안내 텍스트를 배치한다.
7. `MusicSelectionPanel`에는 2~3개의 음악 카드 버튼 자리만 먼저 만든다.
8. `PlayHudPanel`에는 손 인식 상태 텍스트, 제스처 안내 텍스트, 파라미터 게이지 자리만 먼저 만든다.
9. 각 Panel은 나중에 컨트롤러에서 켜고 끌 수 있도록 이름을 명확히 유지한다.
10. 씬을 저장한다.

### 완료 후 체크
- Hierarchy에서 `UE_AppRoot`, `UE_MainCanvas`, `TitlePanel`, `MusicSelectionPanel`, `PlayHudPanel`이 보이는지 확인한다.
- 이 단계는 별도 C# 클래스 Task가 아니므로 Task 체크리스트는 갱신하지 않는다.

## 1단계. 경험 흐름 컨트롤러 배치

### 실행 Task
- `Assets/Tasks/01_UE_ExperienceFlowController.md`

### 서브에이전트 요청 예시
```text
Assets/Tasks/01_UE_ExperienceFlowController.md를 기준으로 UE_ExperienceFlowController를 구현하고 SampleScene에 GameObject로 배치해줘.
의존성 있는 다른 클래스는 구현하지 말고, 필요한 연결 지점은 public 메서드나 직렬화 필드 수준으로만 남겨줘.
완료 후 Assets/Checklists/Tasks/01_UE_ExperienceFlowController_Checklist.md에서 실제 확인한 항목만 체크해줘.
```

### Unity 실습 절차
1. `Assets/Scripts` 또는 프로젝트에서 사용하는 스크립트 폴더를 만든다.
2. `UE_ExperienceFlowController.cs`를 생성한다.
3. `UE_AppRoot` 아래에 빈 GameObject를 만들고 이름을 `UE_ExperienceFlowController`로 지정한다.
4. 생성한 컴포넌트를 GameObject에 붙인다.
5. Inspector에서 초기 상태와 디버그 옵션 기본값을 확인한다.
6. `TitlePanel`, `MusicSelectionPanel`, `PlayHudPanel`을 나중에 제어할 수 있는 구조가 필요한지 확인한다.
7. Unity Console에 컴파일 오류가 없는지 확인한다.
8. 씬을 저장한다.

### 완료 후 체크
- `Assets/Checklists/Tasks/01_UE_ExperienceFlowController_Checklist.md`를 갱신한다.
- Play Mode를 돌리지 않았다면 `Play Mode 미검증`은 체크하지 않는다.

## 4단계. 웹캠과 손 추적 입력 연결

### 실행 Task
- `Assets/Tasks/03_UE_WebcamTrackingController.md`

### MediaPipe 연결 원칙
- 먼저 Mock 손 입력 또는 임시 추적 결과 입력 메서드로 전체 흐름을 만든다.
- MediaPipe Hands는 `UE_WebcamTrackingController` 내부 입력 어댑터로 연결한다.
- MediaPipe Hand Landmark Detection 샘플은 양손 인식을 위해 `NumHands` 기본값을 2로 둔다.
- MediaPipe 원본 랜드마크 타입은 `UE_GestureDjController`나 HUD 쪽으로 직접 전달하지 않는다.
- 제스처 시스템에는 정규화, 좌우 보정, 스무딩, 신뢰도 처리가 끝난 손 상태만 넘긴다.

### 서브에이전트 요청 예시
```text
Assets/Tasks/03_UE_WebcamTrackingController.md를 기준으로 UE_WebcamTrackingController를 구현하고 SampleScene에 GameObject로 배치해줘.
MediaPipe Hands 실제 연동은 교체 가능한 입력 지점으로 만들고, 후속 제스처 클래스는 구현하지 마.
완료 후 Assets/Checklists/Tasks/03_UE_WebcamTrackingController_Checklist.md에서 실제 확인한 항목만 체크해줘.
```

### Unity 실습 절차
1. `UE_WebcamTrackingController.cs`를 생성한다.
2. `UE_AppRoot` 아래에 빈 GameObject를 만들고 이름을 `UE_WebcamTrackingController`로 지정한다.
3. 생성한 컴포넌트를 GameObject에 붙인다.
4. Inspector에서 `UE_ExperienceFlowController` 참조를 연결한다.
5. 웹캠 장치 선택값, 해상도, 좌표 반전, 민감도, 스무딩 기본값을 확인한다.
6. MediaPipe Hands가 아직 없다면 임시 추적 결과 입력 메서드가 있는지 확인한다.
7. Unity Console에 컴파일 오류가 없는지 확인한다.
8. 씬을 저장한다.

### MediaPipe 실제 연결 절차
1. Mock 입력으로 `UE_WebcamTrackingController`가 왼손/오른손 상태를 갱신하는지 먼저 확인한다.
2. Unity 프로젝트에 MediaPipe Hands 패키지 또는 플러그인을 추가한다.
3. MediaPipe가 웹캠 프레임을 받을 수 있도록 `WebCamTexture` 또는 프레임 전달 지점을 연결한다.
4. MediaPipe의 21개 손 랜드마크 결과를 `UE_WebcamTrackingController`의 추적 결과 입력 메서드로 전달한다.
5. 입력 메서드 안에서 랜드마크를 0~1 범위 좌표, 손 구분, 신뢰도, 손목 회전 추정값으로 변환한다.
6. 좌우 반전, 스무딩, 신뢰도 하락 시 마지막 안정값 유지 규칙을 적용한다.
7. `UE_GestureDjController`에는 MediaPipe 원본값이 아니라 안정화된 손 상태 접근자만 연결한다.
8. Play Mode에서 손 미인식, 한 손 인식, 양손 인식, 추적 손실 상황을 각각 확인한다.

### 완료 후 체크
- `Assets/Checklists/Tasks/03_UE_WebcamTrackingController_Checklist.md`를 갱신한다.
- 실제 웹캠 실행과 Play Mode 확인 전이면 관련 항목은 미검증으로 남긴다.

## 최종 확인 절차
1. `Assets/Tasks`에 6개 Task 문서만 있는지 확인한다.
2. `Assets/Checklists/Tasks`에 6개 Task 체크리스트만 있는지 확인한다.
3. `SampleScene` Hierarchy에 `UE_AppRoot`와 6개 `UE_` 시스템 GameObject가 있는지 확인한다.
4. 각 C# 파일이 파일당 하나의 public 클래스 규칙을 지키는지 확인한다.
5. Unity Console 컴파일 오류를 모두 해결한다.
6. Play Mode를 실행했다면 각 체크리스트의 `Play Mode 미검증` 항목을 실제 결과에 맞게 갱신한다.
7. 전체 흐름이 확인되면 `Assets/Checklists/기획문서작업체크리스트.md`의 관련 항목을 갱신한다.
